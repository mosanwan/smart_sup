# 智能桨板项目

## 工作流程

- 做项目级设计或固件修改前，先阅读 `README.md`、`docs/system_overview.md`、`docs/hardware_plan.md` 和 `docs/safety_checklist.md`。
- 硬件和固件修改都按安全敏感变更处理，因为系统使用 7S8P 锂电池组和双大功率无刷推进。
- 用户授权状态是控制目标，不是普通遥测结果：用户手动解锁后，App 层只有三种情况可以把状态改回锁定：用户手动点击锁定、用户语音要求锁定、蓝牙已经断开连接。App 和自动逻辑不得因为普通遥测、导航状态、心跳重建或无故 `ARMED=0` 回传主动锁定；这类反向回传只能记录为扰动并继续维持用户解锁目标。
- 用户手动或语音开启航向锁定后，App 和自动逻辑不得主动退出该锁航状态；只有用户手动取消、用户语音取消、蓝牙断开，或主控明确硬安全故障时才允许退出。没有故障原因的 `HLOCK=OFF` 回传只能记录为扰动并继续维持用户锁航目标。
- App 只负责描述用户和上层导航“要做什么”，例如前进几档、锁定航向、左转 45 度、驶向某个目标航向；主控负责“具体怎么做”，包括航向误差闭环、转向差速、积分/自适应补偿、反打、限幅和最终左右推进器输出。除纯 UI 预览外，App 不得实现或分叉任何推进转向内环算法。
- 左右 ESC 方向反转属于安装/出厂配置，必须记录在 ESP32 主控 NVS，并且只在固件最低层把语义推进百分比映射到实际 PWM 前修正。App 只能在安装/维护设置中写入该配置；App 控制算法、主控闭环算法、高层控制指令、锁航/转向/自动导航算法都不得读取、携带或重复应用 ESC 反向。控制状态和控制日志只记录用户语义 `L/R`，不得暴露底层反向后的 raw 输出。
- 保持 `ref/` 作为外部参考资料目录，用于存放照片、规格书、手册、接线图和测试证据。
- 优先做小而可追踪的设计更新：先在 `docs/` 记录假设，再写入固件。
- 后续对话、文档、注释和项目说明优先使用中文；必要的技术缩写、库名、引脚名和命令可以保留英文。

## 固件默认要求

- ESP32 固件默认保持安全状态：上电未解锁，明确解锁前只输出 ESC 空闲脉宽。
- 没有遥控输入或传感器状态异常时，系统应退回空闲油门或保守限功率。
- 除非先更新硬件方案，不要占用 LoLin32-Lite-LoRa 参考资料中已经用于 LoRa/I2C 的引脚。
- 明确记录 ESC 信号假设：PWM 范围、校准行为、空闲脉宽、最大脉宽和刷新频率。

## 硬件审查

- 推荐更高功率测试前，先检查电芯、BMS、线束、接头、开关、保险、电调和电机的电流限制。
- 未解决的假设要写进文档，不要藏在代码注释里。
- 下水测试必须分阶段：岸上低功率、单电机、双电机、短时下水，再到更长时间和更高功率测试，并记录温度、电压和电流。
