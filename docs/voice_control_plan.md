# Android 云端实时语音 Agent 方案

## 目标

在 Android 控制端加入云端语音 Agent：手机采集用户语音片段，发送给火山方舟 `doubao-seed-2-0-lite-260428` 做音频理解和 Function Call；Android 只执行通过工具 schema、状态校验和限幅后的高层控制事件。语音控制只作为低速辅助输入，不作为高油门或绕过安全联锁的入口。

当前推荐链路：

```text
Android 麦克风
-> 按住说话录音，或实时模式下本地 VAD 自动切分用户语音
-> Android 封装 WAV/base64
-> 火山方舟 Chat API 音频输入
-> doubao-seed-2-0-lite-260428 Function Call / 文本回复
-> Android 按工具 schema、权限和限幅校验
-> 当前控制链路
-> ESP32 固件二次限幅和安全状态机
-> ESC / 电机
-> 文本回复再走 TTS 播放
```

第一版保留按住说话，松开后一次性提交音频给方舟；实时模式采用 Android 本地轻量 VAD 检测人声，只在检测到用户说话后切片上传，不把持续静音直接送给方舟。收到文本回复后先用 Android 系统 TTS 播放。后续再把文本回复接入豆包语音 WebSocket 双向流式 TTS，以降低首包延迟和提升音色质量。

豆包端到端实时语音大模型可以作为“听说体验”参考，但实测其 WebSocket 文本输入没有返回服务端结构化 `tool_calls`，只会把类似 Function Call 的标记作为普通文本流输出，因此不能作为控制面。控制决策必须走火山方舟 Function Calling。

如果网络条件不可用或方舟请求失败，语音 Agent 应进入不可用状态，不自动切回本地 ASR 控制。第一版暂不使用本地 Qwen ASR，避免模型体积、发热和部署复杂度影响验证。

## 实时会话启动和唤醒

不使用本地 ASR 时，不能依赖本地唤醒词。第一版采用两种显式入口：

```text
用户点击 App 的实时语音按钮 / 遥控器语音键
-> Android 开始本地录音
-> 用户松开按键
-> Android 停止录音并提交音频片段给方舟
-> 方舟返回工具调用或文本回复
-> Android 校验后执行，文本回复走 TTS
```

推荐提供两种入口：

| 模式 | 触发方式 | 音频上传 | 用途 |
| --- | --- | --- | --- |
| 按住说话 | 按住 App 按钮或遥控器按键 | 松开后上传单段音频 | 成本最低，适合测试和控制命令 |
| 实时对话 | 点击开启，点击关闭 | 本地 VAD 检测到人声后自动切片上传 | 适合接近实时通话体验，静音不上传 |

实时对话模式必须有明显的前台状态和常驻通知，显示麦克风正在监听、服务商、连接状态、累计时长和估算消耗。默认不允许后台自动开启；如果需要后台运行，必须由用户在前台手动启动前台服务。

当前 Android 本地 VAD 是能量阈值方案，不做本地 ASR：AudioRecord 采集 `16kHz/16-bit/mono` PCM，每 `100ms` 计算 RMS，连续约 `200ms` 超过阈值后认为开始说话，保留约 `300ms` pre-roll；连续约 `900ms` 静音或单段达到 `8s` 后结束切片。短于约 `600ms` 的片段忽略。阈值需要按手机麦克风、外放回声和船上环境继续实测调参。

唤醒词模式下，VAD 切出的音频片段先提交到豆包语音 ASR 极速识别，仅用于判断文本是否以“豆包”开头。未命中唤醒词时，App 不把该音频提交给方舟 LLM、不执行工具、不播报回复；命中后才进入方舟音频 Agent 流程。该模式降低误触发和方舟音频 token 成本，但仍会产生前置 ASR 调用成本。

云端可做服务器侧唤醒词或语音活动检测，但这只用于降低模型推理成本，不能作为隐私或安全边界。App 是否上传音频只由用户显式开关控制。

## 实时双向流

```text
上行音频：
Android AudioRecord
-> PCM16
-> WAV/base64
-> 方舟 Chat API input_audio

下行文本 / 语音：
方舟模型
-> 文本回复
-> Android TTS；后续替换为豆包语音流式 TTS

控制事件流：
方舟模型
-> Function Call / Tool Call
-> Android 按函数名和参数 schema 校验、限幅
-> ESP32 控制心跳
```

关键交互要求：

- 按住说话模式只上传用户主动录制的音频片段，默认不上传环境连续音频。
- 实时对话模式只在用户显式开启后监听；静音阶段只做本地 VAD，不上传音频。
- 单段音频应限制在短时命令长度内，第一版建议 `2-8s`，超过上限自动拒绝或截断。
- 下行语音第一版使用系统 TTS；后续接豆包语音流式 TTS 后再支持边生成边播放。
- 支持 barge-in：用户再次按住说话时中断正在播放的 TTS。
- 使用耳机时优先走耳机麦克风；外放时启用回声消除，避免 Agent 的语音被再次送回模型。
- Android 页面同时显示实时转写、Agent 状态、最后一次控制事件和拒绝原因，便于调试。

## Agent 工具调用

云端模型不能直接输出 PWM、GPIO、电调校准或 ESP32 配置。正式控制入口必须使用 Function Call / Tool Call，而不是靠系统提示词要求模型输出某段 JSON。系统提示词只用于描述角色、安全边界和拒绝策略；是否执行控制以工具调用和 Android 端校验结果为准。

第一版注册九个工具：

| tool name | 允许参数 | 说明 |
| --- | --- | --- |
| `sup_stop` | `reason` | 停止/空挡，只把左右推进器归零，保持当前主控解锁状态，不提供锁主控能力 |
| `sup_set_gear` | `gear`、`reason` | 固定档位：`forward_1..4`、`reverse_1..3`、`neutral` |
| `sup_pivot_turn` | `direction`、`reason` | 原地掉头；`left` 或 `right`。Android 内部先切空挡并等待 `1s`，再用空挡航向锁定执行约 180 度掉头 |
| `sup_set_limited_power` | `left_percent`、`right_percent`、`reason` | 仅用于用户明确要求左右推进器百分比；受声控功率限制和 ESP32 二次限幅 |
| `sup_adjust_heading_target` | `delta_deg`、`base_power_percent`、`duration_ms`、`reason` | 相对角度转向，只用于“左转/右转 X 度” |
| `sup_set_heading_target` | `target_heading_deg`、`base_power_percent`、`reason` | 设置绝对目标航向并持续进入航向锁定 |
| `sup_cancel_heading_lock` | `reason` | 只取消航向锁定，不解锁推进、不关闭保护 |
| `sup_disable_voice_control` | `reason` | 只关闭实时语音/声控会话，不改变推进输出、不锁主控、不取消航向锁定 |
| `sup_explain_status` | `reason` | 只语音回复，不控制推进 |

工具调用示例：

```json
{
  "type": "function_call",
  "name": "sup_set_gear",
  "arguments": {
    "gear": "forward_1",
    "reason": "用户要求低速前进"
  }
}
```

为了兼容调试按钮、自建网关或不支持工具调用的服务端，Android 可以保留旧版 `control_event` 解析，但它只作为兼容格式，不作为正式设计依据：

```json
{
  "type": "control_event",
  "action": "set_gear",
  "gear": "forward_1",
  "reason": "用户要求低速前进"
}
```

Android 每隔 `200-500ms` 向云端发送压缩状态快照，或在状态变化时发送事件，避免模型基于过期状态决策：

```json
{
  "type": "boat_state",
  "armed": true,
  "control_source": "APP",
  "battery_v": 25.8,
  "total_current_a": 12.4,
  "left_percent": 18,
  "right_percent": 18,
  "faults": [],
  "gps_fix": true,
  "phone_heading_deg": 137,
  "heading_age_ms": 120
}
```

云端模型可以说话解释，也可以发工具调用；工具调用和语音回复必须分离。即使模型语音说“我已经停下”，也必须以 `sup_stop` 工具调用成功通过 Android 校验并发送到 ESP32 后，Android 才能在 UI 中显示已进入空挡。锁主控、急停和硬件断电不开放给 Agent，仍由 App 本地按钮、遥控器、硬件急停和 ESP32 保护链路负责。

非控制类语音请求不能硬拒绝成“没有听到有效命令”。用户询问 `有哪些工具`、`支持哪些指令`、`你能做什么`、`怎么使用` 时，Agent 应直接用中文简短说明当前开放的四类能力；只有听不清、无语音或确实没有可回答内容时，才回复没有听到有效命令。

方舟音频 Agent 使用短系统提示词，只声明三件事：直接回应用户语音、不描述音频内容；控制必须调用工具且不能解锁或关闭保护；非控制问题用简短中文回答。提示词明确区分固定档位使用 `sup_set_gear`，停/停止/空挡使用 `sup_stop`，左/右原地掉头使用 `sup_pivot_turn` 且只填写 `direction`，`目标航向/航向设为 X 度` 使用 `sup_set_heading_target`，`左转/右转 X 度` 使用 `sup_adjust_heading_target`，其中左转为负角度、右转为正角度；`取消/退出/解除航向锁定` 使用 `sup_cancel_heading_lock`；`关闭/停止声控` 使用 `sup_disable_voice_control`，不取消锁航。固定提示词后追加每次请求实时生成的状态快照，包括系统时间、主控连接状态、解锁状态、左右推进器目标百分比、左右推进器当前输出百分比、手机航向、航向锁定状态、GPS 模块/定位/卫星/天线、电池和故障字段。ESP32 航向只作为本地影子观测和调试数据，不提供给 Agent。工具 schema 和 Android/ESP32 校验仍是实际安全边界。

为了支持短对话上下文，Android 只保存最近 `5` 条最终给用户看的 Agent 文本，包括普通回复、工具执行成功结果和工具拒绝原因。下一次请求只把这些短文本作为上下文发给方舟，不重复上传历史音频。

工具调用被 Android 拒绝时，不停留在“已拒绝”的 UI 状态；拒绝原因会回写为实时回复并播放，例如 `拒绝执行：语音不能解锁，请先手动解锁`，同时进入最近回复历史，方便用户追问“为什么”。

## 成本和会话控制

- 默认使用按住说话模式验证，确认稳定后再开放实时对话模式。
- 实时对话模式默认单次最长 `3-5min`，超时后需要用户手动继续。
- App 显示本次会话时长、上行音频秒数、下行音频秒数、控制事件数、错误数和估算费用。
- 用户静音超过 `30-60s` 时自动暂停上行音频或提示是否继续。
- 状态快照只发送必要字段，不上传完整日志、轨迹历史或调试文件。
- 日志中默认保存转写文本、控制事件和错误码；原始音频只在采样模式或用户明确开启时保存。

## 本地 ASR 暂缓

本地 Qwen ASR 暂不作为第一版主路径。原因是 Android 本地模型体积接近 1GB，部署、发热、内存和延迟都会影响先验证实时交互体验。

后续如果云端成本、网络可靠性或隐私要求成为瓶颈，再评估本地 Qwen3-ASR-0.6B 作为兜底路径。即使重新启用本地 ASR，它也只负责转文字，不能直接生成油门或绕过控制事件校验。

Android 端在控制事件通过校验、满足连接和解锁条件，并成功发送到 ESP32 后，才允许播放或显示执行成功。实时语音回复可以由云端生成，但不能替代执行结果确认。采样模式、解析拒绝、未连接、未解锁或蓝牙发送失败时不得让 Agent 说出“已执行”。

后台声控如果启用，Android 端必须在用户从前台启动实时语音后开启常驻前台服务，声明 `microphone` 和 `connectedDevice` foreground service type，并显示常驻通知。实时语音会话不能依赖 Activity 是否处于 `STARTED` 状态；退到后台或锁屏后是否继续上传音频必须由用户在设置页明确允许。由于 Android 对麦克风权限有 while-in-use 限制，后台麦克风服务必须在 App 仍处于前台、且已经获得 `RECORD_AUDIO` 权限后启动；如果进程被系统强杀，第一阶段不做后台自启动。

如果后续重新启用本地 Qwen ASR，`sherpa-onnx` 运行时通过 `android/app/libs/sherpa-onnx-1.13.2.aar` 引入，APK 只打包 `arm64-v8a` 原生库。VAD 模型 `silero_vad.onnx` 放在 App assets 中。Qwen ASR 模型不放进 Git；App 优先从自身内部 files 目录读取模型：

```text
/data/data/com.smartsup.controller/files/models/sherpa-onnx-qwen3-asr-0.6B-int8-2026-03-25/
```

需要包含：

```text
conv_frontend.onnx
encoder.int8.onnx
decoder.int8.onnx
tokenizer/merges.txt
tokenizer/tokenizer_config.json
tokenizer/vocab.json
```

下载和推送模型：

```bash
tools/prepare_qwen_asr_model.sh
```

如果脚本检测到手机已通过 `adb devices` 连接，会用 `run-as com.smartsup.controller` 把模型导入 App 内部 files 目录。不要直接 `adb push` 到 `/sdcard/Android/data/...`，shell 创建的目录可能导致 App 侧读取权限不一致。

## 控制意图白名单

第一版只开放低风险控制意图。用户可以用自然语言表达，但云端模型最终必须映射成受限 Function Call / Tool Call，Android 不执行自由文本：

| 语音意图 | 示例说法 | 控制行为 |
| --- | --- | --- |
| 停止/空档 | `停止`、`停下`、`空档`、`空挡`、`回空档` | 保持当前主控解锁状态，把左右目标设为 `0%` |
| 前进一档 | `前进1`、`前进一档`、`慢速前进` | 低速前进 |
| 前进二档 | `前进2`、`前进二档` | 受语音最大油门限制的前进 |
| 前进三档 | `前进三档`、`前进3档` | 使用设置页三档目标，继续受语音最大油门限制 |
| 前进四档 | `前进四档`、`前进4档` | 使用设置页四档目标，继续受语音最大油门限制 |
| 后退一档 | `后退`、`后退1`、`后退一档`、`慢速后退` | 低速后退 |
| 后退二档 | `后退2`、`后退二档` | 使用设置页二档后退目标，继续受语音最大油门限制 |
| 左原地掉头 | `左原地掉头`、`左原地转` | App 内部先切空挡并等待 `1s`，再用空挡航向锁定执行掉头 |
| 右原地掉头 | `右原地掉头`、`右原地转` | App 内部先切空挡并等待 `1s`，再用空挡航向锁定执行掉头 |
| 快点 | `快点`、`快一点`、`快一点点`、`再快一点`、`加速` | 当前左右目标按设置页微调步进加速，默认 `3%` |
| 慢点 | `慢点`、`慢一点`、`慢一点点`、`再慢一点`、`减速` | 当前左右目标按设置页微调步进减速，向 `0%` 收敛 |
| 左转 | `左转`、`往左` | 低速差速左转 |
| 右转 | `右转`、`往右` | 低速差速右转 |
| 左转指定角度 | `左转30度`、`往左十五度` | 未锁航时执行一次角度转向；锁航中把锁航目标左移指定角度 |
| 右转指定角度 | `右转30度`、`往右十五度` | 未锁航时执行一次角度转向；锁航中把锁航目标右移指定角度 |
| 开始声控 | `开始声控`、`开启声控`、`恢复声控` | Android 端恢复执行语音控制命令 |
| 停止声控 | `停止声控`、`关闭声控`、`暂停声控`、`关闭实时语音` | 调用 `sup_disable_voice_control`，只关闭语音会话，不改变推进输出或航向锁定 |
| 保持航向 | `保持航向`、`锁定当前航向`、`方向锁定`、`开启航向锁定` | Android App 记录手机当前指南针航向并进入本地航向锁定 |
| 取消保持航向 | `取消保持航向`、`退出方向锁定`、`取消航向锁定` | 退出航向锁定，回到手动油门/空挡 |

暂不开放：

- `全速前进`
- `最大油门`
- 语音解锁
- Agent 锁主控、硬件急停或关闭保护
- 自由文本直接控制油门、PWM、GPIO 或保护阈值

当前角度转向限制为 `1-90°`。角度命令只设置相对目标，不让语音直接指定任意油门；Android App 用手机指南针本地计算低速差速，并在手机航向失效、超时、失联或系统锁定时取消。

## 限幅规则

语音控制必须同时在 Android 和 ESP32 侧限幅。Android 侧用于用户体验和提前过滤，ESP32 侧作为最终安全边界。

初始固件级限幅：

| 项目 | 初始值 |
| --- | --- |
| 声控功率限制 | 默认 `70%`，Android 设置页可在 `5-100%` 调整，正反向使用同一绝对值上限 |
| 语音左右差速最大差值 | `20%` |
| 语音角度转向最长持续 | `3-5s`，Android 侧实现；档位、空挡和原地掉头不使用持续时间 |
| 语音单次角度转向最大角度 | `90°` |
| ESP32 角度转向超时 | `8s` |
| 航向锁定容差 | 默认 `4°`，App 设置页可配置 |
| 航向锁定最大误差参考 | 默认 `6°`，App 设置页可配置 |
| 航向锁定最大差速修正 | 非空档最高 `70%` |
| 空档原地掉头固定输出 | 默认正推 `28%`、反推 `31%`，App 设置页可配置 |
| 语音航向锁定单侧输出最大值 | 跟随声控功率限制，默认最高 `70%`，可配置到最高 `100%` |
| App 航向锁定单侧输出最大值 | `100%` |

屏幕停止按钮、硬件急停和 ESP32 失联保护不受上述限幅限制，任何状态下都应立即回到空闲油门。云端语音 Agent 不提供锁主控、硬件急停或关闭保护能力；普通语音 `停止` 只作为辅助空挡入口。网络或实时会话异常时，用户必须仍能通过本地按钮、硬件急停或遥控器停止。

`停止声控` 是语音门控命令，不等同于普通 `停止`：执行后 Android 关闭或暂停实时语音会话；恢复需要用户在 App 中重新开启实时语音或按住说话。

`快点` / `慢点` 是相对微调命令，不指定绝对档位。Android 端按设置页“微调步进”调整当前左右目标，默认每次 `3%`，可配置为 `1-10%`；前进时加速提高正向油门，后退时加速提高倒退绝对值，减速向 `0%` 收敛。该命令不能解锁系统，必须在已手动解锁且声控未停止时执行，并以 `SRC=VOICE` 下发，继续受声控功率限制和 ESP32 二次限幅约束。

## 控制命令来源

语音控制发送控制命令时应带来源字段：

```text
SRC=VOICE;ARM=1;L=30;R=30;VMAX=70
SRC=VOICE;ARM=1;L=14;R=-14;VMAX=70
```

手动 App 控制可以省略 `SRC`，默认按普通 App 控制处理；也可以显式发送：

```text
SRC=APP;ARM=1;L=30;R=30
```

ESP32 收到 `SRC=VOICE` 时必须启用语音限幅。Android 会在语音命令中发送 `VMAX` 字段，范围 `5-100`，表示语音来源允许的正反向绝对功率上限；ESP32 侧必须再次夹取该值，缺省按 `70%` 处理。语音命令不能把未解锁系统切换到解锁状态；语音只能在系统已解锁后改变低速目标，或在当前状态下把推进目标归零到空挡。

角度转向和航向锁定不再使用 `MODE=TURN` 或 `MODE=HEADING_LOCK`。Android App 在本地记录手机指南针目标航向并计算左右 ESC 功率，控制心跳始终发送普通 `SRC/ARM/L/R/VMAX`。语音不能在锁定状态下开启航向锁定，必须先由手动方式解锁。取消航向锁定不解锁系统，只退出 App 本地航向闭环；如果没有有效手动油门，输出回到空挡。

## 控制事件校验要求

Android 端不再把自由文本直接解析成控制命令。云端模型可以返回实时转写文本用于显示，但控制只能来自 Function Call / Tool Call。Android 执行前至少检查：

1. 工具调用 schema 合法，函数名必须在白名单内。
2. 参数类型、范围和单位合法；兼容旧版 `control_event` 时也要映射到同一组内部事件。
3. 当前系统状态允许执行，例如已手动解锁、手机航向有效、没有故障、控制链路在线。
4. 单次事件只能表达一个控制动作，不能把多个动作合并绕过限幅。
5. 档位、空挡和原地掉头不接受持续时间；角度转向事件带有 `duration_ms` 时必须小于语音单次动作最长持续时间。
6. 模型语音回复中的自然语言不得作为控制依据。

如果工具调用缺少关键参数、角度超限、功率超限、权限不足、和当前故障状态冲突，Android 端应拒绝执行并保持当前安全状态。拒绝原因需要显示在语音页面，并可发送给云端模型用于下一轮解释。

云端系统提示词必须声明：模型需要控制时只能调用上述工具，不能要求 Android 解析普通文本，也不能编造不在白名单内的动作。但提示词不是安全边界；Android 的工具白名单、参数校验、状态校验和 ESP32 二次限幅才是执行边界。

实时语音模式的额外约束：

- 用户提到 `全速`、`最大油门`、`满油门`、`高速` 等高风险意图时，模型应解释拒绝原因；即使模型错误调用工具，Android 也必须拒绝高风险动作。
- 所有推进命令仍要求系统已由手动方式解锁。
- 实时会话断开、模型超时或工具调用流异常时，Android 不发送新的语音控制命令；已存在的短时语音动作到期后回空闲。
- 云端模型不能通过连续发送短动作来模拟无限时长自动驾驶；Android 需要对连续语音工具调用设置总时长或心跳上限。

角度转向控制策略：

1. Android 解析 `左转xx度` / `右转xx度`，只允许 `1-90°`。
2. Android 以当前手机指南针航向为起点计算相对目标。
3. Android 每个心跳按目标角度误差输出低速差速，接近目标后回空挡。
4. 手机航向不可用、控制心跳超时、控制超时或锁定时取消角度转向。
5. 当前版本是短时相对角度控制，不承担长时间航向保持。
6. 如果航向锁定已经激活，角度命令以当前锁航目标为起点移动目标航向，不退出锁航；即使模型误把 `左转` 输出成正角度，Android 端也会按中文原因纠正为负角度。

航向锁定控制策略：

1. Android 解析 `保持航向` / `方向锁定`，并保留当前左右油门作为两侧独立基础推进油门；语音档位和语音微调可以同时更新两侧基础油门，手动拖动单侧 ESC 只更新该侧基础油门。
2. Android 以当前手机指南针航向作为目标航向。
3. Android 每个心跳计算 `目标航向 - 当前手机航向` 的最短角度误差。
4. 误差在设置页配置的容差内不修正，默认 `4°`；超过容差后按误差大小线性增加差速修正，最小修正为当前 ESC 实测启动门槛 `3%`，默认 `6°` 及以上达到最大修正。当前高响应测试版本最大差速修正为 `70%`，语音来源单侧输出绝对值跟随声控功率限制，手动 App 来源单侧输出最高允许到 `100%`。
5. 有前进基础油门的一侧不主动反推，有后退基础油门的一侧不主动正推；基础油门为 `0%` 的一侧才允许双向原地修正，反推侧默认最多 `-70%`，可在设置页调整。
6. 普通油门、档位、微调和锁航中的语音角度命令不得取消航向锁定，只能更新基础油门或目标航向。
7. 关闭实时语音只停止麦克风会话，不取消已经建立的航向锁定。
8. 手机航向不可用、控制心跳超时、App 本地锁定、硬件急停、界面取消锁航或语音取消航向锁定时取消航向锁定。

实时语音接入计划：

1. 在设置页增加实时语音开关、服务商、API Key、模型、音色、会话超时和费用显示。
2. 先实现按住说话模式：按下建立或复用会话，上传音频流，松开发送输入结束信号。
3. 再实现实时对话模式：持续上传、增量播放、barge-in、前台服务通知和会话超时。
4. 接入结构化控制事件通道，先只允许 `explain_status` 和 `stop`，验证事件解析和执行确认。
5. 逐步开放 `set_limited_power` 和 `adjust_heading_target`，并保留岸上低功率测试门槛。
6. 语音测试页显示连接状态、实时转写、Agent 回复、控制事件、拒绝原因、延迟和估算费用。

## 测试顺序

1. 云端实时语音连通测试：验证上行音频、下行音频、barge-in、断线重连和费用统计。
2. 纯对话测试：只允许 `explain_status`，不允许任何推进控制。
3. 控制事件单元测试：覆盖 schema、越界参数、非法动作、多个动作合并、高风险词和超时拒绝。
4. ESP32 控制命令测试：确认 `SRC=VOICE` 不能解锁，且油门被夹取。
5. 岸上低功率测试：拆桨或架空，验证停止、前进低档、差速转向和模型回复不等于执行确认。
6. 短时下水测试：只开放低档语音控制，并保留手动接管、屏幕停止、遥控器停止和硬件急停。

## 语音样本采集

Android 语音分页提供“采样模式”。开启后 App 可以连接云端实时语音服务并保存转写、回复和控制事件，但控制输出强制保持空闲；识别到语音后只生成待保存样本，不发送蓝牙控制命令。

采样模式流程：

1. 打开“采样模式”开关。
2. 按页面显示的目标短语朗读，例如 `左转三十度`。
3. App 显示本次实时转写、Agent 回复、控制事件和拒绝原因。
4. 如果转写、回复和控制事件符合目标，点 `正确`；否则点 `失败` 保存失败样本。
5. 点 `重录` 会丢弃待保存样本并保留当前目标，点 `下一条` 会切换到下一条目标指令。

样本保存在 App 内部目录：

```text
/data/data/com.smartsup.controller/files/voice_samples/
```

每条样本包含同名 `wav` 和 `json` 文件。`wav` 是本地采集音频或服务商要求的采样格式；`json` 记录目标指令、实时转写、Agent 回复、控制事件、是否被 Android 校验接受、人工标记结果、模型名、延迟、费用估算和时间戳。

当前第一批目标指令覆盖：

- `开始声控`
- `停止声控`
- `停止`
- `空档`
- `前进一档`
- `前进二档`
- `前进三档`
- `前进四档`
- `后退一档`
- `左转`
- `右转`
- `左转十五度`
- `左转三十度`
- `左转六十度`
- `右转十五度`
- `右转三十度`
- `右转六十度`
- `保持航向`
- `锁定当前航向`
- `取消航向锁定`

采样数据先用于实时语音服务评估和控制事件校验优化，不用于直接放宽语音控制权限。重点统计四类指标：首字延迟、语音回复是否自然、控制事件是否正确、Android 是否正确拒绝危险事件；其中误接受比拒绝更危险，应优先排查。
