# Android 本地语音控制方案

## 目标

在 Android 控制端加入本地语音识别，把固定语音术语转换成受限控制命令。语音控制只作为低速辅助输入，不作为高油门或绕过安全联锁的入口。

当前推荐链路：

```text
Android 麦克风
-> AudioRecord 16kHz 单声道 PCM
-> silero_vad.onnx 语音活动检测
-> sherpa-onnx + Qwen3-ASR-0.6B int8 ONNX
-> 文本规范化和白名单模式匹配
-> 语音控制限幅
-> 经典蓝牙 SPP
-> ESP32 固件二次限幅和安全状态机
-> ESC / 电机
```

如果网络条件可用，后续可以把 ASR 层扩展为“在线优先、本地兜底”：

```text
Android 麦克风
-> VAD / 端点检测
-> 在线 ASR（已配置密钥且网络可用）
-> 失败或超时后切回本地 Qwen ASR
-> 文本规范化、候选命令打分和白名单解析
-> 语音控制限幅
```

在线模型只能提高转写质量，不能直接输出油门。无论文本来自在线 ASR 还是本地 Qwen ASR，都必须进入同一个确定性解析器和同一套安全限幅。

## 本地 ASR 选择

优先评估 Qwen3-ASR-0.6B 作为 Android 本地语音转文字模型。选择 0.6B 的原因是比 1.7B 更适合移动端验证，内存、发热和延迟风险更低。

Qwen ASR 只负责语音转文字，不能直接生成油门或控制命令。控制命令必须由确定性的白名单解析器生成。

当前 Android App 的“语音”分页已经切换为本地 Qwen ASR 后端：申请录音权限后自动连续监听，用 `sherpa-onnx` 的 `OfflineRecognizer` 加载 Qwen3-ASR-0.6B int8 ONNX 模型，并用 VAD 切分语音段。页面会显示 ASR 运行状态、ASR 转写文本、解析后的控制命令和执行结果；识别到的最终文本会送入同一个白名单解析器和语音限幅路径。

语音分页还会显示候选命令和匹配分数。解析器会先做文本规范化和常见 ASR 误识别修正，再对固定指令集打分；只有最高候选超过阈值，且与第二候选拉开足够差距时才允许执行。分数不足、候选接近、同时包含多个动作、出现高油门词、角度词或后退高档时，Android 端拒绝发送控制命令。

Android 端在语音命令通过白名单解析、满足连接和解锁条件，并成功发送到 ESP32 后，使用系统 TTS 播放简短确认。普通成功命令可从 `好的`、`可以`、`明白了`、`收到`、`yes sir`、`好的，帅哥` 等短句中随机选择；停止、锁定、声控开关和航向锁定类命令仍应保留明确状态词，例如 `收到，已停止`、`明白了，航向已锁定`。角度转向可随机回复纯确认或带角度信息，例如 `好的`、`左转30度`、`已锁定34度`。采样模式、解析拒绝、未连接、未解锁或蓝牙发送失败时不播放成功确认，避免给用户造成命令已执行的错觉。TTS 播放开始后到播放完成后 `1s` 内，Android 端必须暂停 ASR 识别输入、丢弃麦克风采样并清空 VAD/ASR 缓冲；这段时间不更新语音预览、不执行解析结果、不发送控制命令。如果 TTS 回调异常，使用最长 `15s` 的保险屏蔽窗口，避免确认语音本身被连续监听误触发成下一条控制命令。

后台声控是刚需：Android 端必须在用户从前台启动声控后开启常驻前台服务，声明 `microphone` 和 `connectedDevice` foreground service type，并显示常驻通知。ASR 监听不能再依赖 Activity 是否处于 `STARTED` 状态；退到后台或锁屏后仍需继续监听。由于 Android 对麦克风权限有 while-in-use 限制，后台麦克风服务必须在 App 仍处于前台、且已经获得 `RECORD_AUDIO` 权限后启动；如果进程被系统强杀，第一阶段不做后台自启动。

`sherpa-onnx` 运行时通过 `android/app/libs/sherpa-onnx-1.13.2.aar` 引入，APK 当前只打包 `arm64-v8a` 原生库。VAD 模型 `silero_vad.onnx` 放在 App assets 中。Qwen ASR 模型体积接近 1GB，不放进 Git；App 优先从自身内部 files 目录读取模型：

```text
/data/data/com.smartsup.controller/files/models/sherpa-onnx-qwen3-asr-0.6B-int8-2026-03-25/
```

需要包含：

```text
conv_frontend.onnx
encoder.int8.onnx
decoder.int8.onnx
tokenizer/merges.txt
tokenizer/tokenizer_config.json
tokenizer/vocab.json
```

下载和推送模型：

```bash
tools/prepare_qwen_asr_model.sh
```

如果脚本检测到手机已通过 `adb devices` 连接，会用 `run-as com.smartsup.controller` 把模型导入 App 内部 files 目录。不要直接 `adb push` 到 `/sdcard/Android/data/...`，shell 创建的目录可能导致 App 侧读取权限不一致。

## 语音命令白名单

第一版只开放低风险固定命令：

| 语音意图 | 示例说法 | 控制行为 |
| --- | --- | --- |
| 停止 | `停止`、`停下`、`急停`、`锁定` | 发送 `ARM=0;L=0;R=0` |
| 空档 | `空档`、`空挡`、`回空档` | 在已解锁状态下保持解锁并把左右目标设为 `0%` |
| 前进一档 | `前进一档`、`慢速前进` | 低速前进 |
| 前进二档 | `前进二档` | 受语音最大油门限制的前进 |
| 前进三档 | `前进三档`、`前进3档` | 使用设置页三档目标，继续受语音最大油门限制 |
| 前进四档 | `前进四档`、`前进4档` | 使用设置页四档目标，继续受语音最大油门限制 |
| 后退一档 | `后退一档`、`慢速后退` | 低速后退 |
| 快点 | `快点`、`快一点`、`快一点点`、`再快一点`、`加速` | 当前左右目标按设置页微调步进加速，默认 `3%` |
| 慢点 | `慢点`、`慢一点`、`慢一点点`、`再慢一点`、`减速` | 当前左右目标按设置页微调步进减速，向 `0%` 收敛 |
| 左转 | `左转`、`往左` | 低速差速左转 |
| 右转 | `右转`、`往右` | 低速差速右转 |
| 左转指定角度 | `左转30度`、`往左十五度` | 未锁航时执行一次角度转向；锁航中把锁航目标左移指定角度 |
| 右转指定角度 | `右转30度`、`往右十五度` | 未锁航时执行一次角度转向；锁航中把锁航目标右移指定角度 |
| 开始声控 | `开始声控`、`开启声控`、`恢复声控` | Android 端恢复执行语音控制命令 |
| 停止声控 | `停止声控`、`关闭声控`、`暂停声控` | Android 端立即发送锁定/空挡，并忽略后续命令直到 `开始声控` |
| 保持航向 | `保持航向`、`锁定当前航向`、`方向锁定`、`开启航向锁定` | ESP32 记录当前 IMU 航向并进入持续航向锁定 |
| 取消保持航向 | `取消保持航向`、`退出方向锁定`、`取消航向锁定` | 退出航向锁定，回到手动油门/空挡 |

暂不开放：

- `全速前进`
- `最大油门`
- `后退二档` 和更高后退档
- 语音解锁
- 自由文本直接控制油门

当前角度转向限制为 `1-90°`。角度命令只设置相对目标，不让语音直接指定任意油门；ESP32 仍按语音限幅输出低速差速，并在 IMU 异常、超时、失联或系统锁定时取消。

## 限幅规则

语音控制必须同时在 Android 和 ESP32 侧限幅。Android 侧用于用户体验和提前过滤，ESP32 侧作为最终安全边界。

初始固件级限幅：

| 项目 | 初始值 |
| --- | --- |
| 声控功率限制 | 默认 `70%`，Android 设置页可在 `5-100%` 调整，正反向使用同一绝对值上限 |
| 语音左右差速最大差值 | `20%` |
| 语音单次动作最长持续 | `3-5s`，Android 侧实现 |
| 语音单次角度转向最大角度 | `90°` |
| ESP32 角度转向超时 | `8s` |
| 航向锁定容差 | 默认 `4°`，App 设置页可配置 |
| 航向锁定最大误差参考 | 默认 `6°`，App 设置页可配置 |
| 航向锁定最大差速修正 | `70%` |
| 空档锁航最大反推 | 默认 `70%`，App 设置页可配置 |
| 语音航向锁定单侧输出最大值 | 跟随声控功率限制，默认最高 `70%`，可配置到最高 `100%` |
| App 航向锁定单侧输出最大值 | `100%` |

`停止`、`急停` 和 `锁定` 不受上述限幅限制，任何状态下都应立即回到空闲油门。

`停止声控` 是语音门控命令，不等同于普通 `停止`：执行后 Android 继续监听并显示 ASR 文本，但只响应 `开始声控`，其他语音控制命令全部忽略。

`快点` / `慢点` 是相对微调命令，不指定绝对档位。Android 端按设置页“微调步进”调整当前左右目标，默认每次 `3%`，可配置为 `1-10%`；前进时加速提高正向油门，后退时加速提高倒退绝对值，减速向 `0%` 收敛。该命令不能解锁系统，必须在已手动解锁且声控未停止时执行，并以 `SRC=VOICE` 下发，继续受声控功率限制和 ESP32 二次限幅约束。

## 蓝牙命令来源

语音控制发送蓝牙命令时应带来源字段：

```text
SRC=VOICE;ARM=1;L=30;R=30;VMAX=70
SRC=VOICE;ARM=1;MODE=TURN;DIR=LEFT;ANGLE=30;TID=12;LREV=0;RREV=0;VMAX=70
SRC=VOICE;ARM=1;MODE=HEADING_LOCK;HLOCK=1;BASE=20;HTOL=4;HFULL=6;HREV=70;HID=7;LREV=0;RREV=0;VMAX=70
```

手动 App 控制可以省略 `SRC`，默认按普通 App 控制处理；也可以显式发送：

```text
SRC=APP;ARM=1;L=30;R=30
```

ESP32 收到 `SRC=VOICE` 时必须启用语音限幅。Android 会在语音命令中发送 `VMAX` 字段，范围 `5-100`，表示语音来源允许的正反向绝对功率上限；ESP32 侧必须再次夹取该值，缺省按 `70%` 处理。语音命令不能把未解锁系统切换到解锁状态；语音只能在系统已解锁后改变低速目标，或在任何状态下执行停止/锁定。

未锁航时，角度转向命令使用 `MODE=TURN`。Android 每次新的角度语音命令生成递增 `TID`，ESP32 只在看到新的 `TID` 时记录当前 IMU 航向并设置目标角度；同一 `TID` 的重复命令只作为心跳，避免连续心跳反复重置目标。

锁航已激活时，角度语音命令不使用 `MODE=TURN`，而是继续发送 `MODE=HEADING_LOCK`，使用新的 `HID` 和一次性 `HOFF` 调整锁航目标。`HOFF` 范围 `-90..90`，负值为左转，正值为右转；Android 首次发送成功后必须把后续心跳改成同一 `HID`、不带 `HOFF` 的保持命令，ESP32 也必须按 `HID` 防止同一 `HOFF` 重复消费。锁航中语音前进/后退、快点/慢点只更新 `BASE`，不能取消锁航。

航向锁定命令使用 `MODE=HEADING_LOCK`。Android 每次开启航向锁定生成递增 `HID`，ESP32 只在看到新的 `HID` 时把当前 IMU 航向记录为目标；同一 `HID` 的重复命令只作为心跳。`HTOL` 和 `HFULL` 来自 Android 设置页，分别控制航向锁定容差和达到最大差速修正的误差角；`HREV` 控制 `BASE=0` 空档锁航时允许的最大反推。语音不能在锁定状态下开启航向锁定，必须先由手动方式解锁。取消航向锁定不解锁系统，只退出航向闭环；如果没有有效手动油门，输出回到空挡。

## 文本解析要求

Android 端不能用简单 `contains("前进")` 作为最终判断，应至少分三步：

1. 文本规范化：去标点、去空格、把中文数字转换为阿拉伯数字。
2. 同义词归一：例如 `左转`、`左拐`、`往左` 统一为 `TURN_LEFT`。
3. 白名单完整匹配：只有匹配到单个明确意图时才发控制命令。

如果识别结果同时包含多个动作、缺少关键参数、角度超限或置信度不足，Android 端应拒绝执行并保持当前安全状态。

当前 App 已实现候选命令打分，覆盖常见识别偏差，例如 `左传` 归一为 `左转`、`右专` 归一为 `右转`、`前进一段` 归一为 `前进一档`、`到车` 归一为 `倒车`。这类归一只服务于固定白名单命令，不能扩展成自由文本控制。

无唤醒词时的额外约束：

- 只接受短固定术语，不接受长句里的 `contains` 命中。
- 最高候选分数不足时拒绝执行。
- 最高候选和第二候选分数接近时拒绝执行。
- 识别到 `全速`、`最大油门`、`满油门`、`高速` 等高风险词时硬拒绝；前进三档和前进四档允许作为固定档位命令，但仍受声控功率限制和 ESP32 二次限幅。
- 停止 / 锁定可以在任何状态执行；所有推进命令仍要求系统已由手动方式解锁。
- `停止声控` 后只响应 `开始声控`，其他命令仅显示转写和候选，不发送控制命令。

角度转向控制策略：

1. Android 解析 `左转xx度` / `右转xx度`，只允许 `1-90°`。
2. ESP32 收到新 `TID` 后，以当前 IMU Z 轴积分航向为起点计算相对目标。
3. ESP32 在控制周期中按目标角度误差输出低速差速，接近目标后回空挡。
4. IMU 不可用、读取失败、蓝牙心跳超时、控制超时或锁定时取消角度转向。
5. 当前版本是短时相对角度控制，不承担长时间航向保持；后续需要磁力计/GPS 修正漂移。
6. 如果航向锁定已经激活，角度命令只移动锁航目标，不退出锁航。

航向锁定控制策略：

1. Android 解析 `保持航向` / `方向锁定`，并用当前左右油门平均值作为基础推进油门。
2. ESP32 收到新 `HID` 后，以当前 IMU Z 轴积分航向作为目标航向。
3. ESP32 每个控制周期计算 `目标航向 - 当前航向` 的最短角度误差。
4. 误差在设置页配置的容差内不修正，默认 `4°`；超过容差后按误差大小线性增加差速修正，默认 `6°` 及以上达到最大修正。当前高响应测试版本最大差速修正为 `70%`，语音来源单侧输出绝对值跟随声控功率限制，手动 App 来源单侧输出最高允许到 `100%`。
5. 有前进基础油门时，默认只做“外侧加推、内侧降推”，不主动让一侧反推；基础油门为 `0%` 时才允许双向原地修正，反推侧默认最多 `-70%`，可在设置页调整。
6. 普通油门、档位、微调和锁航中的语音角度命令不得取消航向锁定，只能更新基础油门或目标航向。
7. IMU 不可用、读取失败、蓝牙心跳超时、锁定、急停、界面取消锁航或语音取消航向锁定时取消航向锁定。

在线 ASR 接入计划：

1. 在设置页增加在线 ASR 开关、服务商、密钥和超时配置。
2. VAD 切出的语音段优先发在线 ASR；超时、网络错误或鉴权失败时自动用本地 Qwen ASR。
3. 语音测试页显示当前 ASR 后端、转写文本、候选命令和拒绝原因。
4. 采集测试日志，比较本地 Qwen、在线 ASR 和混合策略在风噪、水声、手机防水壳内的命令级成功率。

## 测试顺序

1. 离线录音测试：收集安静环境、风噪、水声、不同距离下的命令音频。
2. Android 端 ASR 延迟测试：记录识别耗时、内存、温度和耗电。
3. App 解析器单元测试：覆盖同义词、中文数字、误识别和拒绝执行场景。
4. ESP32 蓝牙命令测试：确认 `SRC=VOICE` 不能解锁，且油门被夹取。
5. 岸上低功率测试：拆桨或架空，验证停止、前进低档和差速转向。
6. 短时下水测试：只开放低档语音控制，并保留手动接管和硬件急停。

## 语音样本采集

Android 语音分页提供“采样模式”。开启后 App 会保持 ASR 连续监听，但控制输出强制保持空闲；识别到语音后只生成待保存样本，不发送蓝牙控制命令。

采样模式流程：

1. 打开“采样模式”开关。
2. 按页面显示的目标短语朗读，例如 `左转三十度`。
3. App 显示本次 ASR 文本和解析结果。
4. 如果 ASR 和解析结果符合目标，点 `正确`；否则点 `失败` 保存失败样本。
5. 点 `重录` 会丢弃待保存样本并保留当前目标，点 `下一条` 会切换到下一条目标指令。

样本保存在 App 内部目录：

```text
/data/data/com.smartsup.controller/files/voice_samples/
```

每条样本包含同名 `wav` 和 `json` 文件。`wav` 是 16kHz 单声道 PCM 音频；`json` 记录目标指令、ASR 文本、解析命令、是否被解析器接受、人工标记结果、模型名和时间戳。

当前第一批目标指令覆盖：
