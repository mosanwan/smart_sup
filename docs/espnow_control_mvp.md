# ESP-NOW 遥控 MVP

## 目标

第一版遥控器使用 `ESP32-C3 SuperMini`，通过 `ESP-NOW` 直接向桨板主控 ESP32 发送控制心跳，先跑通“绑定、解锁、油门、急停、失联回空闲”的最小闭环。

该方案替代经典蓝牙 SPP 作为第一版实时遥控链路。Android App、语音控制、GPS 轨迹同步和 OTA 后续可以保留为维护/辅助链路，但不得代替主遥控心跳。

## 链路选择

| 项目 | 当前约定 |
| --- | --- |
| 遥控器硬件 | ESP32-C3 SuperMini |
| 桨板主控 | ESP32，需支持 ESP-NOW |
| 无线链路 | ESP-NOW，2.4GHz |
| 控制频率 | 遥控器每 `20ms` 发送一次，实际有效控制按 `50Hz` 处理 |
| 失联阈值 | 主控超过 `300-500ms` 无有效控制帧时进入锁定/空闲 |
| 加密 | 启用 ESP-NOW peer LMK，加密密钥写入两端配置 |
| Wi-Fi 模式 | `WIFI_STA`，固定信道，禁止运行时自动切换信道 |

ESP-NOW 不需要路由器，也不需要经典蓝牙配对。首次绑定可通过 USB 串口写入对端 MAC，也可以后续增加短按绑定按钮流程。第一版建议先用 USB 串口固定写入，减少误绑定风险。

## 遥控器输入

ESP32-C3 SuperMini 作为手持遥控器，第一版建议只保留必要输入：

| 输入 | 用途 | 安全要求 |
| --- | --- | --- |
| 左/右摇杆或油门+转向 | 生成左右 ESC 目标 | 上电校准中位，超出合理 ADC 范围视为异常 |
| 解锁开关/按键组合 | 明确解锁 | 单个误触按键不得直接解锁 |
| 急停/锁定按键 | 发送 `ARM=0;L=0;R=0` | 任意状态立即生效 |
| 档位/限速按键 | 限制最大油门 | 不能绕过主控侧限幅 |
| 遥控器电池采样 | 低电提醒 | 遥控器低电时主控可降级或拒绝解锁 |

摇杆、按键和电池采样的具体 GPIO 需要按实际 SuperMini 板卡引脚确认后再写入 `docs/hardware_plan.md` 和固件。不要在未确认启动绑定位、ADC 能力和板载 LED/USB 占用前直接定死引脚。

## ESP32-C3 SuperMini 遥控器引脚

当前第一版按“双摇杆分别控制左右 ESC”设计，不做油门/转向混控。左摇杆直接生成左侧 ESC 目标，右摇杆直接生成右侧 ESC 目标。

| 功能 | 建议 GPIO | 接法 | 说明 |
| --- | --- | --- | --- |
| 左 ESC 摇杆 ADC | GPIO0 | 左摇杆中间端接 GPIO0，两端接 `3V3/GND` | ADC1；上电校准中位，断线或超范围进入 `ESTOP` |
| 右 ESC 摇杆 ADC | GPIO1 | 右摇杆中间端接 GPIO1，两端接 `3V3/GND` | ADC1；上电校准中位，断线或超范围进入 `ESTOP` |
| 遥控器电池 ADC | GPIO3 | 电池经分压后接 GPIO3 | ADC1；分压后不得超过 `3.3V` |
| 备用 ADC | GPIO4 | 暂不接 | 可留给第二油门轴、电位器限速或后续传感 |
| 急停 / 锁定 | GPIO5 | `GPIO -- 按键 -- GND`，固件 `INPUT_PULLUP` | 按下立即发送 `ARM=0` 和 `ESTOP=1` |
| 解锁按键 | GPIO6 | `GPIO -- 按键 -- GND`，固件 `INPUT_PULLUP` | 需要长按或组合动作，不能单次误触解锁 |
| 减档 / 限速降低 | GPIO7 | `GPIO -- 按键 -- GND`，固件 `INPUT_PULLUP` | 降低 `vmax`，最低回空挡/锁定 |
| 加档 / 限速提高 | GPIO10 | `GPIO -- 按键 -- GND`，固件 `INPUT_PULLUP` | 只提高遥控器请求上限，主控仍二次限幅 |
| 蜂鸣器 / 震动提示 | GPIO20 | GPIO 驱动三极管或 MOS 管 | 不要直接用 GPIO 带大电流蜂鸣器/震动马达 |
| 状态 LED | GPIO21 | 串联限流电阻接 LED | 显示上电、链路、低电和故障状态 |

暂不使用 `GPIO2`、`GPIO8`、`GPIO9` 作为第一版控制输入。这几个是 ESP32-C3 启动绑定位，外部按键、摇杆或模块在复位瞬间把它们拉到错误电平，可能导致不能正常启动或不能刷写。`GPIO8` 也常被部分 SuperMini 板卡接到板载 LED，具体以实物丝印和万用表确认。

`GPIO20/21` 在使用 USB CDC 调试时可以作为普通 GPIO；如果后续改用硬件 UART 调试或外接串口模块，需要重新评估这两个引脚。

## 控制帧

ESP-NOW 使用二进制控制帧，避免依赖字符串解析。所有多字节字段使用小端序。

```text
magic      uint16  固定 0x5355，表示 "SU"
version    uint8   当前 1
type       uint8   1=control, 2=bind, 3=telemetry
remote_id  uint16  遥控器编号
seq        uint32  单调递增序号
flags      uint16  bit0=ARM, bit1=ESTOP, bit2=CALIBRATING
left       int8    左 ESC 目标，-100..100
right      int8    右 ESC 目标，-100..100
vmax       uint8   遥控器侧最大油门限制，5..100
remote_mv  uint16  遥控器电池电压，单位 mV；未知填 0
crc16      uint16  覆盖 crc16 之前的所有字段
```

主控收到控制帧后必须检查：

- `magic/version/type` 正确。
- 来源 MAC 与已绑定遥控器一致。
- `seq` 没有明显倒退；重复包可以丢弃。
- `crc16` 正确。
- `left/right/vmax` 在合法范围内。
- 遥控器未声明 `ESTOP`，且主控本地安全条件允许解锁。

`ARM=1` 只表示遥控器请求解锁。主控仍要同时满足本地低电压、传感器、温度、看门狗和急停条件，才允许 ESC 输出高于空闲脉宽。

## 回传帧

主控可以用 ESP-NOW 低频回传状态，第一版建议 `2-5Hz`：

```text
magic      uint16
version    uint8
type       uint8   3=telemetry
controller_id uint16
seq        uint32
armed      uint8
fault      uint16
left_out   int8
right_out  int8
batt_mv    uint16
total_a_x10 uint16
rssi_hint  int8    无可用值时填 0
crc16      uint16
```

回传只用于遥控器显示状态和蜂鸣提醒，不作为主控安全判断依据。没有回传时，遥控器应提示失联；主控仍以收到有效控制帧的时间为失联保护依据。

## 主控安全状态

- 上电默认未解锁，ESC 输出保持空闲脉宽。
- 收到有效控制帧前不得解锁。
- 控制帧超时 `300-500ms` 进入锁定/空闲。
- 收到 `ESTOP` 或 `ARM=0` 立即回空闲。
- 遥控器请求油门仍要经过主控侧限幅、斜率限制、低电压/过流/过温保护。
- 遥控器断电、重启或序号回退时，主控不得保持上一帧油门。

## 测试顺序

1. USB 串口打印双方 MAC，固定写入 peer 配置。
2. 桌面短距离测试 ESP-NOW 收发、序号、CRC 和失联超时。
3. 不接 ESC，只看主控串口日志，验证解锁、锁定、急停和摇杆中位。
4. 接 ESC 但拆桨/架空，验证空闲脉宽、低油门和斜率限制。
5. 岸上低功率单电机测试，再双电机测试。
6. 短时下水低功率测试，记录失联距离、天线摆放、温度、电压和电流。

2.4GHz 在水面、人体遮挡、电池和金属结构附近会明显衰减。遥控器和主控天线应尽量远离大电流线、电池包、碳纤/金属件，并在防水结构外侧保留射频窗口。
