## ESC PWM 假设和上电时序

当前固件按双向 RC PWM 处理 ESC 信号：

| 项目 | 当前值 |
| --- | --- |
| PWM 频率 | `50Hz` |
| 最大后退 | `1000us` |
| 空闲 / 中位 | `1500us` |
| 最大前进 | `2000us` |
| 控制刷新周期 | `20ms` |
| 油门斜率限制 | 每周期最多变化 `5us`，约 `25%/s` |
| 最低有效推进 | 实测绝对值小于 `9%` 时电机不转，固件按空闲脉宽处理 |
| 上电预备窗口 | ESP32 PWM 初始化后先保持 `1500us` 空闲 `3s`，再初始化传感器和蓝牙 |

如果实物 ESC 是单向航模电调，空闲/解锁脉宽可能是 `1000us` 而不是 `1500us`。在确认 ESC 说明书和低功率实测前，不要把当前固件直接切到单向映射，因为对双向 ESC 来说 `1000us` 可能等价于最大后退。现场排查时优先看 Android 控制页的主控回传：若 `ARMED=1` 且 `LPWM/RPWM` 已随油门从 `1500us` 变化，但电机仍不响应，重点检查 ESC 校准方式、上电蜂鸣状态、ESP32 与 ESC 信号地共地、信号线接触和 ESC 是否支持当前双向 PWM 范围。

2026-06-20 岸上低功率实测：当前电机/ESC 组合约 `9%` 推进才开始缓慢转动，`1%..8%` 可视为空闲死区。固件在最终百分比转 PWM 前应用该死区，绝对值小于 `9%` 的左右推进命令保持 `1500us` 空闲脉宽；`9%` 及以上仍按原线性 PWM 映射和斜率限制输出。下水后需要复测起转阈值，水中负载可能提高最低有效推进百分比。

## IMU 接线计划

参考图片：`ref/IMU.jpg`，原计划模块型号为 ICM20948V2 九轴传感器。当前新增一块带内部姿态解算的九轴 IMU 模块，优先使用 I2C 方式连接。

| ICM20948V2 引脚 | ESP32 建议连接 | 说明 |
| --- | --- | --- |
| VCC | 3V3 | 优先使用 3.3V，避免 5V 逻辑风险 |
| GND | GND | 必须与 ESP32、电调信号地共地 |
| SDA | GPIO14 | I2C 数据线；接 ESP32 丝印 `R16/14` |
| SCL | GPIO22 | I2C 时钟线；状态 LED 已改用 GPIO2 |
| ADO | GND | I2C 地址通常为 `0x68` |
| INT | 暂不接 | 后续需要中断/数据就绪时再接 |
| NCS | 3V3 | 使用 I2C 时保持片选无效；若模块已上拉可不接 |
| FSYNC | 暂不接 | 外部同步输入，当前不用 |
| EDA/ECL | 暂不接 | 辅助 I2C 总线，当前不用 |

注意：LoLin32-Lite-LoRa 参考扩展板里 I2C 使用 GPIO12/GPIO14，但 GPIO12 是 ESP32 启动绑定位，外接上拉可能影响启动。当前实物 ESP32 板未引出 GPIO21，直连 IMU 时使用 GPIO14/GPIO22，并在固件里显式 `Wire.begin(14, 22)`。

2026-06-19 新九轴 IMU 接线后，用 ESP32 在 `GPIO14/GPIO22`、`100kHz` 下扫描到 I2C 地址 `0x23`。该地址不符合裸 ICM20948 常见地址 `0x68/0x69`，结合资料中的 `IMU_I2C_ReadEuler()`、`IMU_I2C_ReadQuaternion()` 等函数，判断该模块更可能通过板载 MCU 输出已融合姿态数据；后续固件应按模块通信协议单独接入，不复用当前裸 ICM20948 寄存器读取逻辑。上线初期默认只做姿态日志和 Android 对比；Android 设置页可手动开启“航向锁定使用 ESP32 IMU”测试模式，把 `YBY` 用于航向锁定和角度转向，自动导航仍不使用该 IMU 航向。

同日补充资料 `通信协议.xlsx` 截图显示的是串口通信帧格式，不是裸 I2C 寄存器表。帧格式为 `0x7E 0x23 LEN FUNC DATA... CHECKSUM`，`LEN` 为整帧长度，`CHECKSUM` 为从包头 1 累加到校验位前一字节后取最低 8 位；多字节数据低位在前。已看到的命令包括请求固件版本 `FUNC=0x80`、返回固件版本 `FUNC=0x01`、IMU 校准 `FUNC=0x70`、返回状态 `FUNC=0x81`。若只接 `SDA/SCL`，该串口帧协议不能直接用于 ESP32 当前接线；如果后续改用 UART，需要重新规划 ESP32 串口引脚并更新接线图。

只读 I2C 探针进一步确认新模块存在低地址数据区，均为小端格式：

| 地址 | 长度 | 数据 | 换算 |
| --- | --- | --- | --- |
| `0x04` | 6 | 三轴加速度 `int16` | `raw * 16 / 32767`，单位 `g` |
| `0x0A` | 6 | 三轴角速度 `int16` | `raw * 2000 / 32767 * pi / 180`，单位 `rad/s` |
| `0x10` | 6 | 三轴磁场 `int16` | `raw * 800 / 32767`，单位 `uT` |
| `0x16` | 16 | 四元数 `float[4]` | 顺序 `w, x, y, z` |
| `0x26` | 12 | 欧拉角 `float[3]` | 单位 `rad`；日志显示时可乘 `57.2957795` 转为角度 |

当前静止实测示例：加速度约 `0.10g, -0.10g, 0.99g`，四元数模长约 `1.0`，欧拉角约 `-5.6deg, -5.7deg, 31deg`。旧 10 轴寄存器表中的 `0x34/0x37/0x3D` 在该模块上读数为 0，不作为后续驱动依据。

## GPS 接线计划

参考图片：`ref/GPS1.jpg`、`ref/GPS2.jpg`。模块丝印可见 `VCC/GND/TX/RX/PPS`，当前已用 ESP32 验证可用，且 GPS 的 RX 不需要连接。

| GPS 引脚 | ESP32 建议连接 | 说明 |
| --- | --- | --- |
| VCC | 3V3 或 5V/VIN | 以模块已验证供电为准；若不确定，优先 3.3V |
| GND | GND | 必须与 ESP32 共地 |
| TX | GPIO16 | GPS 串口输出，接 ESP32 的 RX2 |
| RX | 暂不接 | 当前只读取定位数据，不需要向 GPS 发配置 |
| PPS | GPIO13 | 秒脉冲，用于确认 GPS 模块工作和后续时间同步 |

建议固件中使用 ESP32 的第二串口读取 GPS。当前固件会从 `9600` 开始扫描常见 GPS 波特率，找到有效 NMEA 后锁定该波特率：

```cpp
Serial2.begin(9600, SERIAL_8N1, 16, -1);
```

其中 `16` 是 ESP32 接收 GPS TX 的引脚，`-1` 表示不使用 ESP32 TX。PPS 使用 `GPIO13` 作为普通数字输入读取上升沿，不向 GPS 模块输出信号。

当前实测该 GPS 模块 NMEA 输出波特率为 `115200`。模块可以输出 `$BDRMC`、`$BDGGA`、`$BDGSA`、`$BDGSV`、`$BDGLL`、`$BDZDA` 和 `$GPTXT` 等语句；若日志出现 `ANTENNA OPEN`、`RMC` 状态为 `V`、`GGA` fix quality 为 `0` 或卫星数为 `00`，表示串口数据正常但尚未定位，需检查天线/室外视野。

## 自动导航传感器分工

| 模块 | 主要用途 | 注意事项 |
| --- | --- | --- |
| GPS | 位置、对地速度、对地航向、航点距离 | 低速或静止时航向不稳定，更新率可能只有 1Hz |
| IMU 陀螺仪 | 中期融合航向影子观测 | 只能提供角速度，需要零偏校准和融合算法 |
| IMU 加速度计 | 中期融合航向影子观测 | 用于估计重力方向和倾斜补偿，不能直接给出航向 |
| IMU 磁力计 | 中期融合航向影子观测 | 不再裸算航向参与控制，需要 3D 校准、倾斜补偿和异常降权 |
| 双电机差速 | 转向执行 | 左右电机输出差异越大，转向越明显 |

当前航向保持闭环移到 Android App：手机实时读取指南针航向，App 记录锁航目标、计算航向误差和左右 ESC 功率，ESP32 只执行最终 `L/R` 功率命令。正式下水前必须保留人工接管和急停。

手机指南针方案要求手机固定在桨板上，且手机朝向相对船体保持不变；手机拿在手里、放入口袋或被乘员转动时不得用于航向锁定。该方案不能替代硬件急停、人工接管和低功率分阶段测试。

新九轴 IMU 测试模式只影响 Android App 本地的航向锁定和角度转向：App 使用 ESP32 蓝牙 `STATUS` 中约 `10Hz` 上报的 `YBY`，并在 `500ms` 内未收到有效 IMU 航向时退出锁航。该模式需要在设置页显式打开，切换来源会取消当前锁航并回空挡；自动导航仍要求手机指南针可用，不跟随该开关。

ICM20948 中期融合航向方案见 [ICM20948 融合航向中期方案](imu_fusion_heading_plan.md)。该方案第一阶段只作为影子航向源上报并与手机航向对比，不改变自动导航和航向锁定的默认控制来源。

## 电量显示模块接线计划

参考图片：`ref/battery_indicator.png`。该模块用于本地显示电池大致电量，通常只有电源正负两根线或正负焊盘，不需要接 ESP32。

| 电量显示模块 | 建议连接 | 说明 |
| --- | --- | --- |
| 正极 / `+` | 电池总正极，建议接在主开关和保险之后 | 只在系统上电后显示，避免长期耗电 |
| 负极 / `-` | 电池总负极 | 与整车电源负极共地 |

注意事项：

- 必须确认模块支持 7S 三元锂电池，也就是满电约 29.4V、标称约 25.9V 的电压范围。
- 如果模块是“12V/3S/4S”版本，不能直接接 7S 电池，会烧坏。
- 不建议直接长期并在电池包两端，因为显示模块会有静态耗电，长时间放置可能慢慢耗电。
- 推荐接在主开关之后；如果需要随时查看电量，可以单独加一个防水小按钮，让按下时才点亮显示。
- 这个模块只能显示粗略电量，不能替代 BMS，也不能替代 ESP32 的低电压保护采样。

## DC-DC 降压模块接线计划

参考图片：`ref/dc_dc_6-32v_to_5v.png`。该模块标称输入 6-32V、输出 5V，可用于从 7S 电池母线降压，为 ESP32 和低压电子系统供电。

| DC-DC 引脚 | 建议连接 | 说明 |
| --- | --- | --- |
| `IN+` | 主开关和保险之后的电池正极 | 7S 满电约 29.4V，在模块 32V 标称输入范围内 |
| `IN-` | 电池总负极 | 与 ESP32、ESC 信号地共地 |
| `OUT+` | ESP32 的 `5V` / `VIN` | 不要接 ESP32 的 `3V3` |
| `OUT-` | ESP32 的 `GND` | 与系统地相连 |

建议电源路径：

```text
7S 电池正极 -> 保险 -> 主开关 -> DC-DC IN+
7S 电池负极 ---------------------> DC-DC IN-

DC-DC OUT+ -> ESP32 5V/VIN
DC-DC OUT- -> ESP32 GND
```

注意事项：

- 接 ESP32 前，先用万用表确认 `OUT+` 和 `OUT-` 之间确实是 5V 左右。
- 不要把 5V 输出接到 ESP32 的 `3V3` 引脚。
- IMU 等 3.3V 传感器优先接 ESP32 的 `3V3`，不要直接接 DC-DC 的 5V。
- GPS 如果已验证可用，可按原测试供电；不确定时优先 3.3V。
- DC-DC 模块需要防水、防震和固定，输入侧建议加小保险或与低压电子支路共用独立保险。
- 电机大电流线和 DC-DC/ESP32 线束尽量分开走线，降低电磁干扰。

## 双路电流传感器接线计划

参考图片：`ref/acs758lcb_050_current_sensor.png`。计划使用 2 个 ACS758LCB-050 霍尔电流传感器，分别检测左右 ESC 的直流输入电流。

传感器应串在每个 ESC 的直流输入支路上，不要串到电机三相线上。

推荐高边接法：

```text
主电源正极 -> 左电流传感器 IP+ -> 左电流传感器 IP- -> 左 ESC 粗红线
主电源正极 -> 右电流传感器 IP+ -> 右电流传感器 IP- -> 右 ESC 粗红线

主电源负极 -----------------------------------------------> 左/右 ESC 粗黑线
```

低压信号接法：

| ACS758 模块引脚 | ESP32 建议连接 | 说明 |
| --- | --- | --- |
| VCC | ESP32 `3V3` | 推荐 3.3V 供电，避免输出超过 ESP32 ADC 量程 |
| GND | ESP32 `GND` | 与系统地共地 |
| 左侧 OUT | GPIO36 | 只输入 ADC1，引脚也可能标为 `OU1` |
| 右侧 OUT | GPIO39 | 只输入 ADC1，引脚也可能标为 `OU1` |
| OUT2 / OU2 | 暂不接 | 若模块提供第二路缓冲输出，先不用 |

电流限幅策略：

- 左右两路电流分别测量，固件计算 `总电流 = 左侧电流 + 右侧电流`。
- BMS 标称最大放电 60A，软件限流建议先保守设为 45-50A。
- 单侧 50A 传感器不代表单侧可以长期跑 50A；还要受 BMS、线束、接头、ESC 散热和电机负载限制。
- 当总电流超过限制时，优先按比例降低左右油门；如果某一侧单独过流，可以只降低该侧油门。

采样和标定：

- ACS758LCB-050B 属于双向版本时，零电流输出约为 `VCC/2`。3.3V 供电时，零点约 1.65V。
- 5V 供电时典型灵敏度约 40mV/A；3.3V 供电时输出按供电比例变化，约为 26.4mV/A。
- 上电后、ESC 未解锁且电机不转时，采样一段时间作为零点偏移。
- 方向如果接反，电流读数可能为负；可以交换 IP+ / IP-，也可以在固件里取反。
- ESP32 ADC 噪声较大，建议多次采样平均，并在传感器 VCC/GND 旁边加去耦电容。

## 应急有线控制器计划

参考图片：`ref/wired_controller_joystick.png`。计划作为无线遥控链路失效时的备用控制器，平时不接或不启用，插入后由 ESP32 读取两个单轴摇杆和多个按键。

控制优先级建议：

```text
急停/硬件断电 > 有线控制器 > ESP-NOW 遥控器 > 自动导航
```

两个单轴摇杆用于控制两路电机：

| 控件 | 建议用途 | 说明 |
| --- | --- | --- |
| 左摇杆 | 左电机油门/反向 | 中位附近为停止，前推为正转，后拉为反转 |
| 右摇杆 | 右电机油门/反向 | 中位附近为停止，前推为正转，后拉为反转 |

注意：电机正反转需要 ESC 支持双向模式。如果当前 100A ESC 只支持单向油门，则摇杆后拉只能作为刹车/停止，不能真正反转。是否支持反转必须看 ESC 说明书或实测确认。

单轴摇杆通常是 `10K` 电位器，图中 `103` 表示 10kΩ。每个摇杆按三线电位器接法：

```text
摇杆一端 -> ESP32 3V3
摇杆另一端 -> ESP32 GND
摇杆中间滑动端 -> ESP32 ADC 输入
```

建议先用万用表确认哪一个脚是中间滑动端：摇杆移动时，中间端到两端的电阻会一边变大、一边变小。

四个按键暂定功能：

| 按键 | 建议功能 | 行为 |
| --- | --- | --- |
| 加档 | 提高当前档位或限速档 | 短按上调一档，不能绕过软件限幅 |
| 减档 | 降低当前档位或限速档 | 短按下调一档，最低回到空档/停止 |
| 回到空档 | 立即回到空闲油门 | 短按生效，左右推进输出回到空闲脉宽 |
| 方向锁定 | 保持当前航向 | 需要 IMU 正常可用，手动摇杆明显动作时应取消 |

按键建议接法：

```text
ESP32 GPIO -- 按键 -- GND
```

固件里启用内部上拉 `INPUT_PULLUP`，按下时读到低电平。这样每个按键只需要一根信号线加公共地线。

有线控制器建议连接线束：

| 线 | 用途 |
| --- | --- |
| 3V3 | 两个摇杆电位器供电 |
| GND | 摇杆和按键公共地 |
| 左摇杆模拟量 | 接 ESP32 ADC |
| 右摇杆模拟量 | 接 ESP32 ADC |
| 按键 1-4 | 接 ESP32 数字输入 |

最低线数为 8 芯：

```text
1 根 3V3
1 根 GND
2 根摇杆模拟信号
4 根按键信号
= 8 根
```

推荐购买 10 芯或 12 芯信号线：

- 10 芯：满足当前方案，并多 2 根备用线。
- 12 芯：更推荐，后续可加控制器插入检测、指示灯、蜂鸣器、解锁键或备用按键。

线规建议：

- 有线控制器只传低压信号和极小电流，`26AWG` 已经够用。
- 如果线缆较长、经常插拔、要穿防水接头或需要更结实，优先选 `24AWG`。
- 如果长度超过 2 米，建议用屏蔽多芯线，或者至少让摇杆模拟信号远离电机/电调大电流线。
- 屏蔽层建议只在 ESP32 舱一端接 GND，遥控器端悬空，减少地环路。
- 不要让有线控制器线缆承载电机、电调或电池主电流。

当前不使用 LoRa 时，ADC1 引脚基本够用：GPIO34 用于电池电压，GPIO36/GPIO39 用于两路电流，GPIO35/GPIO32 用于两个摇杆，GPIO33 保留给温度或后续扩展。若后续增加更多模拟量，建议考虑 I2C ADC 扩展芯片，例如 ADS1115/ADS1015。

安全要求：

- 有线控制器插入后，不应自动解锁；当前线控方案不提供启动/解锁按键。
- 有线控制器断开、摇杆信号异常、ADC 超范围或按键线断开时，系统回到空闲油门。
- 摇杆中位需要设置死区，避免轻微偏移导致电机缓慢转动。
- 连接器必须防水、防呆、防拉脱；建议使用航空插头或防水多芯插头。
- 有线控制器只能作为软件控制备用，不能替代硬件急停。
