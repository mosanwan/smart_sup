# 自动导航路线和执行方案

## 目标

第一版自动导航用于低速航点跟随、航迹线锁定和定点保持：用户在 Android App 地图上维护路线，选择路线后由 App 根据设置页选择的 GPS 来源计算目标点、目标航向和基础推进，并持续发送给 ESP32。GPS 来源可选 `ESP32 GPS` 或 `手机 GPS`；默认仍为 ESP32 GPS。手机 GPS 主要用于主控未连接时在导航页测试地图实时位置、路线点和手机方向显示；如果用于实际自动导航，仍必须连接主控、手动解锁，并通过 App 的定位有效性检查。快速航向稳定内环由 ESP32 固件负责，固件使用主控 IMU `YBY/YBGZ` 计算最终左右功率。导航页蓝色箭头可以显示手机指南针或主控 IMU 的参考方向，但启动固件锁航/自动航向稳定前必须确认主控 IMU 航向和实际船头方向一致。ESP32 不保存路线、不独立规划路径，并保留未解锁、失联、低压、过流、过温和故障回空闲保护。

自动导航只能作为低速辅助功能，不替代人工观察、人工接管和硬件急停。

## 安全边界

- 自动导航不得自动绕过上电默认锁定；执行前必须由用户在 App 中明确确认。
- App 必须持续发送控制心跳；ESP32 收不到心跳时回空闲油门。
- 自动导航的路线规划和目标航向由 Android 负责，快速航向稳定由 ESP32 固件负责；固件锁航要求主控 IMU 安装方向固定且 `YBY/YBGZ` 读数有效。主控 IMU 失效或超过超时阈值时，必须停止自动导航并回空闲。
- 导航页蓝色箭头与实际船头方向不一致时，后续航向误差和推进修正均不可信；应先切换到读数可信的手机或 IMU 航向源。
- GPS 未定位、卫星数不足或手机 GPS 精度不足、坐标缺失、路线少于 2 点、距离目标异常跳变时，App 必须拒绝执行或停止执行。
- 自动导航基础推进档位为 `15%..70%`，高档位只用于逐步验证后的开阔水面测试；最终左右输出仍受自动导航 `70%` 上限和设置页最大油门限制。
- 定点保持第一阶段只允许低速辅助泊车，最大自动位置推进限制为 `30%`，并保留 `12%` 最低有效输出以避开当前电机起转死区；实际输出仍受设置页最大油门限制。
- 手动停止按钮必须始终可见，并立即发送 `ARM=0;L=0;R=0` 或至少 `L=0;R=0` 的保守命令。
- 手动遥控、有线控制器、急停和 ESP32 保护优先级高于自动导航。

推荐控制优先级：

```text
硬件急停 / 断电 > 有线控制器 > ESP-NOW 遥控器 > Android 手动控制 > Android 自动导航
```

## Android 功能

当前 Android Debug 版已实现第一版 App 端路线功能：

- 路线保存在手机本地 `files/navigation_routes/routes.json`。
- 导航页“路线”入口可添加、编辑、删除和执行路线。
- 路线编辑模式支持地图点击添加航点、点击编号航点选中、删除选中航点、把选中航点移动到下一次地图点击位置、前移/后移航点顺序、撤销最后航点、保存和取消；手动缩放地图后继续添加航点不会强制回到默认缩放。
- 自动导航执行时显示当前速度、目标航点、到目标距离、航向误差、当前档位、左右输出、加减档和停止按钮。
- 自动导航启动成功后，App 必须通过 TTS 播报“开始自动驾驶，请注意安全”，提醒用户保持人工接管准备。
- 导航页“定点”按钮可记录当前 GPS 位置和目标航向，进入定点保持；执行时显示当前速度、目标距离、前后误差、左右误差、目标航向、航向误差、当前档位、左右输出、加减档和停止按钮。
- 导航页顶部实时位置、速度和地图蓝色航向箭头跟随设置页的 GPS 来源；轨迹同步仍只同步 ESP32 主控缓存的历史轨迹。
- 自动导航仍复用 App 控制心跳，ESP32 不保存路线、不独立规划；涉及航向稳定时使用固件锁航命令发送目标航向和基础推进。

### 路线入口

导航页新增“路线”按钮，点击后展开路线列表。

路线列表显示：

- 路线名称。
- 航点数量。
- 估算总距离。
- 最近修改时间。
- 操作：执行、编辑、删除。

### 路线编辑模式

新增路线后进入路线编辑模式：

- 地图点击添加航点；手动缩放后继续点击添加，不重置地图相机。
- 航点按添加顺序连成路线，并在地图上显示编号。
- 点击编号航点后可删除、前移、后移，或点击“移动”后用下一次地图点击设置新位置。
- 可保存、取消。
- 保存前至少需要 `2` 个航点。

第一版暂不做手势拖拽，避免和地图拖动/缩放冲突；航点位置调整通过“移动后点击地图”完成。后续再增加插入点、重命名和路线复制。

### 执行模式

用户选中路线后点击“执行”，进入自动驾驶界面：

- 地图显示路线、当前目标航点和当前位置。
- 显示当前速度。
- 显示目标航点编号、到目标距离、航向误差。
- 显示当前自动导航档位。
- 提供加档、减档和停止按钮。

执行前检查：

- 已连接 ESP32。
- GPS `GPS_FIX=1`。
- 卫星数满足最低要求，建议 `GPS_SAT >= 4`。
- 当前实时坐标可用。
- 设置页选择的航向源可用。
- 路线至少 `2` 个点。

## 控制策略

第一版使用“指向目标航点”的简单闭环：

1. 启动时只在路线航点中查找距离当前位置最近的航点，初始目标就是这个最近航点；执行过程中只按航点编号顺序推进，不再按路线段投影跳点。
2. 对自动导航使用的 GPS 坐标做轻量低通滤波；如果坐标跳变超过异常阈值，重置滤波点。
3. 用滤波后的 GPS 坐标计算目标方位 `target_bearing` 和目标距离 `distance_m`。
4. 未进入到达半径前，目标方位始终取“当前位置到当前目标航点”的方位，不提前切到下一段路线方向。
5. App 将目标方位 `target_bearing` 作为固件锁航目标航向，并给出自动导航档位对应的基础推进。
6. 航向误差较小时，固件锁航在基础推进上叠加小差速；航向误差过大时，固件先用受限差速原地转向到目标方位。
7. 左右差速修正由 ESP32 固件锁航内环根据 `YBY/YBGZ` 计算；大角度转向允许一侧正推、一侧反推，但仍受自动导航转向上限、设置页最大油门和固件安全门控限制。
8. 到达当前目标航点半径内后切到下一个航点。
9. 到达最后航点后自动停止。

建议初始参数：

| 参数 | 初始值 | 说明 |
| --- | --- | --- |
| 到达半径 | `8m` | GPS 低速漂移较大，半径不宜太小 |
| GPS 滤波系数 | `0.35` | 新坐标权重，降低低速漂移对控制的影响 |
| GPS 跳变重置阈值 | 默认 `5m`，设置范围 `3m..30m` | 超过该距离视为异常跳变或重定位，重置滤波 |
| 基础档位 | `15% / 22% / 30% / 40% / 50% / 60% / 70%` | App 自动导航档位 |
| 自动导航输出上限 | `70%` | 基础推进和差速修正后的单侧最终输出都不得超过该上限，且仍受设置页最大油门限制 |
| 最大修正 | `25%` | 左右差速最大修正 |
| 原地转向阈值 | `30°` | 航向误差超过该值时先低速原地转向，不向目标点平移 |
| 原地转向输出 | `25%` | 原地转向时的单侧输出上限，仍受设置页最大油门限制 |
| GPS/航向短暂无效 | 暂停输出并等待恢复 | 不自动锁定退出；GPS/航向恢复后继续当前目标航点 |
| 满修正角度 | `45°` | 大误差时达到最大修正 |
| GPS 最低卫星数 | `4` | 低于该值不执行 |
| 手机 GPS 可用条件 | 最近 `5s` 内有位置且精度不差于 `25m` | 手机源没有 ESP32 的卫星数字段，满足条件时按等效通过最低定位检查 |
| 航向超时 | 手机 `1.5s`，主控 IMU `500ms` | 所选航向源无更新则退出 |
| 坐标超时 | `3s` | GPS 坐标无更新则退出 |

差速示例：

```text
base = 当前档位基础推进百分比
correction = clamp(error / full_error, -1, 1) * max_correction

left = base + correction
right = base - correction
```

如果误差超过 `30°`，App 下发低基础推进或空档目标，由 ESP32 固件先用受限原地转向把船头转向目标点；转入 `30°` 内后再恢复向目标点推进。GPS 或主控 IMU 短暂无效时，系统暂停输出并保留自动导航状态；恢复后继续当前目标航点，不因为定位状态抖动自动锁定退出。

## 航迹线锁定第一阶段

航迹线锁定是单段路线跟踪模式，用于水流或侧风造成“板头方向正确但 GPS 轨迹逐渐偏离”的场景。第一阶段的路线、横向偏差和 LOS 目标航向仍在 Android App 中计算，不新增 ESP32 路线协议；ESP32 接收目标航向和基础推进后，用固件锁航内环计算左右推进功率，并保留失联、限幅、低压、过流、过温和故障保护。

启动时 App 记录：

- `origin`：当前 GPS 位置。
- `line_bearing`：当前所选航向源读数，作为目标航迹线方向。
- `gearIndex`：沿用自动导航低速档位，初始为第二档；如果现场推力过大，可在界面降档。

每个控制心跳周期：

1. 读取并滤波当前 GPS 点；GPS 跳变超过设置页阈值时重置滤波。
2. 计算当前点到目标航迹线的有符号横向偏差 `cross_track_error_m`，右侧为正、左侧为负。
3. 使用 LOS 引导计算纠偏角：

```text
correction_angle = atan2(-cross_track_error_m, lookahead_m)
target_heading = line_bearing + clamp(correction_angle, -max_angle, +max_angle)
```

4. 用所选航向源计算 `heading_error = shortest(target_heading - heading)`。
5. 沿用自动导航的低速基础档位，把目标航向和基础推进下发给 ESP32 固件锁航内环。

第一阶段参数：

| 参数 | 初始值 | 说明 |
| --- | --- | --- |
| 前视距离 `lookahead_m` | `12m` | 横向偏差转为航向修正的尺度 |
| 最大纠偏角 | `25°` | 防止 GPS 抖动导致大角度转向 |
| 最大横向偏差 | `15m` | 超出后停止航迹线锁定并回空闲 |
| 基础档位 | `15% / 22% / 30% / 40% / 50% / 60% / 70%` | 复用自动导航档位；最终输出仍受自动导航 `70%` 上限和设置页最大油门限制 |
| 最大差速修正 | `25%` | 复用自动导航修正上限 |
| GPS 最低卫星数 | `4` | 低于该值不执行 |
| 航向超时 | 手机 `1.5s`，ESP32 IMU 测试模式 `500ms` | 复用现有航向源约束 |

主控九轴 IMU 航向模式只使用模块内置算法输出的欧拉角 `YBY`。当前实物安装为 `+X` 朝船头，固定采用 `YBY + 180°` 并归一化到 `0..360°` 作为船头航向。设置页只允许在手机和主控 IMU 两个航向源之间切换；不再提供手机船头偏置、IMU 零点偏置或 `YBY 原始/反向` 算法选择。转换后的航向用于地图箭头、航迹线锁定和航向误差计算。

安全边界：

- 航迹线锁定不得自动解锁；启动前必须已连接、已手动解锁、GPS 定位、卫星数满足要求、航向源有效。
- 用户停止、GPS 无效、航向无效、横向偏差过大或目标航向误差超过 `120°` 时，App 必须停止推进或锁定回空闲。
- 该模式只能作为低速辅助；测试顺序仍按岸上模拟、岸上低功率、静水低速短时、逐步扩大距离执行。

## 定点保持第一阶段

定点保持用于“GPS 锚定”：点击导航页“定点”后，Android 记录当前 GPS 坐标和目标船头航向。之后 App 在每个控制心跳里根据 GPS 位置误差计算回点目标航向和基础推进，ESP32 固件锁航内环根据主控 IMU 计算最终左右推进输出，并保留失联和故障保护。

位置估计第一阶段采用弱融合：GPS 仍是唯一位置来源，主控 IMU 加速度只作为“GPS 跳变是否可信”的短时运动线索。App 不对加速度做二次积分，不用它直接推算坐标，避免低成本 IMU 零偏和姿态误差在几秒内积累成错误位置。

第一阶段控制结构：

```text
目标点 - 当前 GPS -> 距离和目标点方位
目标点方位 / 目标航向 - 主控 IMU 航向 -> 航向误差
三段式控制器 -> 先转向目标点 / 直线推进 / 到点后调整目标航向
目标航向 + 基础推进 -> ESP32 固件锁航内环 -> L/R 功率
```

启动时记录：

- `target_point`：当前 GPS 坐标。
- `target_heading`：当前手机指南针航向。
- `gearIndex`：沿用自动导航低速档位，初始为第三档；如果现场推力过大，可在界面降档。

每个控制心跳周期：

1. 读取当前 GPS 点，并读取主控 `IAX/IAY/IAN` 作为运动线索。
2. GPS 未出现明显跳变时，按常规低通滤波位置。
3. GPS 单次跳变超过设置页阈值时，如果 IMU 运动线索很弱且当前没有明显推进输出，则降低该 GPS 点权重，而不是立刻重置到新点。
4. GPS 跳变同时伴随明显 IMU 运动或推进输出时，接受该跳变并提高 GPS 权重。
5. 计算滤波位置到目标点的距离和方位。
6. 距离超过重新进入半径时，计算“前进回点方位”和“倒车回点方位”，选择当前转向角度更小的一种；方向选择带滞回，避免 GPS 抖动时前进/倒车反复跳变。
7. 船头与所选回点方位误差小于阈值后，才给基础推进；前进回点用正推，倒车回点用负推。
8. 到达保持半径内后停止位置推进。
9. 在目标点附近只修小角度目标航向误差；目标航向误差过大时不做大角度原地转向，避免 360 度绕圈。
10. 前后误差和左右误差仍写入日志，但不参与第一版决策。

第一阶段参数：

| 参数 | 初始值 | 说明 |
| --- | --- | --- |
| 位置保持半径 | `2.0m` | 进入该范围后停止位置推进 |
| 重新进入回点半径 | `2.8m` | 形成滞回，避免在 2.5m 附近反复启停 |
| 满输出距离 | `6m` | 距离达到该值时使用当前定点档位的完整基础推进 |
| 最大位置推进 | `30%` | 仍受设置页最大油门限制 |
| 最低有效位置推进 | `12%` | 避开当前电机起转死区，并给水中负载留余量 |
| 允许推进航向误差 | `20°` | 未基本朝向目标点前只转向，不平移 |
| 前进/后退选择滞回 | `15°` | 倒车方向必须比前进方向少转至少该角度才选择倒车 |
| 到点后航向修正上限 | `45°` | 超过该误差时不做原地大角度调头 |
| 原地航向修正最大差值 | `50%` | 定点原地修正比普通空档锁航略保守 |
| GPS 最低卫星数 | `4` | 低于该值不执行 |
| 当前航向源超时 | 主控 IMU `500ms`；手机 `1.5s` | `YBHDG/YBGZ` 或 `PHDG` 无有效更新则停止自动导航推进 |

安全边界：

- 定点保持不得自动解锁；启动前必须已连接、已手动解锁、GPS 定位、卫星数满足要求、当前航向源有效。
- 定点保持使用 ESP32 固件锁航内环；Android 不对 IMU 加速度做位置积分，也不在 App 端计算最终左右差速。
- GPS 或主控 IMU 短暂无效时，App 必须暂停定点保持输出并等待恢复，不清除定点目标；用户停止、通信中断或目标点无效时才停止定点保持并回空闲。
- 定点保持依赖 GPS 静止精度；岸上或室内无定位时不得测试推进，只能看 UI 拒绝和状态提示。
- 该模式必须按岸上模拟、岸上低功率、静水低速短时、逐步扩大偏离距离测试。

### 定点决策日志

每次启动定点保持时，Android 会在手机外部应用目录创建一个新的 CSV 日志：

```text
Android/data/com.smartsup.controller/files/station_keeping_logs/station_keeping_<time>.csv
```

导航页定点面板会显示当前日志路径和样本数。每个控制心跳记录一行，主要字段包括：

- 当前 GPS、目标 GPS、卫星数、GPS 速度。
- 原始 GPS、滤波 GPS、GPS 跳变量、GPS 权重、融合原因。
- 主控 IMU 加速度 `IAX/IAY/IAZ/IAN` 和短时运动线索强度。
- 目标航向、当前主控 IMU 航向、当前决策航向、航向误差和 `YBGZ`。
- 到目标距离、前后误差、左右误差。
- 是否正在回点、是否倒车、当前档位、输出限幅。
- 基础推进、差速修正、App 视角左右输出、实际下发左右命令。
- 主控上一条 `STATUS` 中的 `ARMED/L/R/LPWM/RPWM` 和原始状态行。

后续调参时优先用该日志判断：

1. `left_output_pct/right_output_pct` 是否过大导致围绕目标震荡。
2. `distance_m` 和 `lateral_error_m` 是否被 GPS 噪声放大。
3. `heading_error_deg` 是否持续大幅滞后。
4. App 输出和主控 `LPWM/RPWM` 是否一致，排除主控限幅、未解锁或 ESC 死区问题。

## App 到 ESP32 控制

第一版复用 App 控制命令心跳，不新增 ESP32 路线协议。普通手动推进仍可发送最终左右功率；自动导航需要航向稳定时，App 周期性发送固件锁航目标航向和基础推进：

```text
SRC=APP;ARM=1;MODE=HEADING_LOCK;HLOCK=1;HID=<id>;BASE=<base>;TARGET=<heading>;VMAX=70
```

停止时发送：

```text
SRC=APP;ARM=0;L=0;R=0;H_SRC=PHONE
```

ESP32 侧仍按现有规则处理：

- 未解锁或故障时不输出推进。
- 控制心跳超时回空闲。
- 输出受限幅、斜率、低压/过流/过温保护约束。

后续如果要让 ESP32 独立执行路线，必须另起设计：路线校验、持久化、航点状态机、独立 GPS/IMU 闭环、人工接管和安全审查都需要在 ESP32 侧实现。

## 测试阶段

1. App 纯 UI 测试：添加、删除、保存路线。
2. 回放/模拟位置测试：用模拟 GPS 点验证航点切换和左右输出计算，但不连接推进。
3. 岸上空载测试：ESP32 接收自动导航命令，但 ESC 输出保持空闲或断开电机。
4. 岸上低功率单电机测试。
5. 岸上低功率双电机测试。
6. 静水低速短时测试，人工随时接管。
7. 逐步扩大路线距离和档位。

任何阶段出现 GPS 漂移、手机方向不稳定、通信延迟、意外转向或停止不及时，都必须退回上一阶段。
