# 安全检查清单

## 上电前

- 电池组绝缘、极性、总压、单串电压一致性检查完成。
- 主电源串联合适保险或熔断保护。
- 急停开关可直接切断推进功率，且位置易触达。
- ESP32、传感器、ESC 信号地共地，电源路径清晰。
- 推进器周围无人员、绳索、工具和松动物件。

## 固件保护

- 上电默认未解锁。
- 解锁前 ESC 输出保持空闲脉宽。
- 油门变化有斜率限制。
- 遥控信号超时进入空闲油门。
- 电池低压进入限功率或空闲油门。
- 传感器异常进入保守状态。
- 语音控制不能解锁系统，只能在已解锁状态下输出受限低速目标。
- 语音控制必须在 ESP32 侧二次限幅，不能只依赖 Android App 限幅。
- 语音前进三档/四档只能作为固定档位目标，仍必须受声控功率限制和 ESP32 二次限幅约束；声控功率默认 `70%`，可配置到最高 `100%`，提高前必须按岸上限流和低功率水下测试逐级验证。
- Android TTS 播报开始到播报完成后 `1s` 内必须暂停 ASR 识别输入、丢弃麦克风采样并清空 VAD/ASR 缓冲，不得把 TTS 确认语音再次解析成控制命令。
- `快点` / `慢点` 只能按设置页微调步进调整当前目标，不能绕过手动解锁、声控功率限制或 ESP32 侧限幅。
- 语音角度转向必须依赖 Android 手机指南针正常读取；未锁航时手机航向失效、角度控制超时或控制心跳丢失必须取消并回空挡。
- 航向锁定中语音角度转向只能移动锁航目标，普通油门、档位和微调只能更新基础油门，不能取消航向锁定。
- 航向锁定必须依赖 Android 手机指南针正常读取；手机航向失效、控制心跳丢失、锁定、急停、界面取消锁航或语音取消航向锁定时必须取消。
- 当前航向显示和锁航计算不得使用 ESP32 IMU 磁力计或陀螺仪积分；Android App 必须使用手机指南针航向，并在 App 本地记录锁航目标、计算误差和最终左右 ESC 功率。
- 手机指南针是唯一航向来源，且手机必须固定在桨板上、相对船体方向不变；手机拿在手里、放入口袋或被乘员转动时不得用于航向锁定。手机航向超过 `1.5s` 无有效更新时，App 必须退出角度转向/航向锁定并回空挡或锁定。
- 航向锁定保留可配置角度容差、最大转向角度和空档最大反推，避免指南针抖动导致持续修正或空档原地修正过猛；当前高响应测试版本超过容差后的最小差速修正为 `3%`，最高 `70%`，空档反推默认最多 `-70%`，手动 App 来源单侧输出可到 `100%`，只能按岸上限流、低功率、短时下水的顺序逐步验证。
- 自动导航不得让 ESP32 独立执行未审查路线；第一版必须由 Android App 计算最终左右功率并持续发送心跳，ESP32 继续执行失联回空闲、限幅和故障保护。
- 自动导航执行前必须检查 GPS 已定位、卫星数满足最低要求、手机航向可用、路线至少 2 个点；任一条件失效必须停止自动导航并回空闲。
- 自动导航界面必须始终显示停止按钮；停止按钮必须立即发送空闲或锁定命令，不能只退出界面。
- `停止声控` 后 Android 端不得执行除 `开始声控` 之外的语音命令。
- 看门狗开启，重启后仍然保持未解锁。

## 岸上测试

- 先使用限流电源或串联保险测试。
- 单电机分别测试方向和油门范围。
- 双电机低油门联动测试。
- 记录电池线、ESC、接头、电机温度。

## 水下测试

- 静水低功率短时测试。
- 每次测试后检查进水、胶封、接头和温升。
- 逐步增加油门和持续时间。
- 保留测试日志，记录电压、电流、温度、故障和环境。
