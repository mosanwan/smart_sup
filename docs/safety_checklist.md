# 安全检查清单

## 上电前

- 电池组绝缘、极性、总压、单串电压一致性检查完成。
- 主电源串联合适保险或熔断保护。
- 急停开关可直接切断推进功率，且位置易触达。
- ESP32、传感器、ESC 信号地共地，电源路径清晰。
- 推进器周围无人员、绳索、工具和松动物件。
- ESC 上电后能识别空闲脉宽；Android 控制页主控回传 `ARMED/L/R/LPWM/RPWM` 与预期一致后，才允许接桨或下水测试。

## 固件保护

- 上电默认未解锁。
- 解锁前 ESC 输出保持空闲脉宽。
- ESP32 PWM 初始化后先保持 ESC 空闲脉宽至少 `3s`，避免电调上电握手窗口内收到不稳定信号。
- 油门变化有斜率限制。
- 遥控信号超时进入空闲油门。
- 电池低压进入限功率或空闲油门。
- 传感器异常进入保守状态。
- 语音控制不能解锁系统，只能在已解锁状态下输出受限低速目标。
- 语音控制必须在 ESP32 侧二次限幅，不能只依赖 Android App 限幅。
- LLM 实时语音 Agent 只能在用户显式开启按住说话或实时对话会话后上传麦克风音频流；不得后台自动开启或静默上传。
- 唤醒词模式下，没有“豆包”前缀的音频片段不得提交给方舟 LLM，也不得执行任何控制工具；只能用于前置 ASR 判断并被丢弃。
- 屏幕停止按钮、遥控器停止、硬件急停和 ESP32 失联保护必须本地可用，不得等待 LLM、网络或云端 ASR 返回；云端语音 `停止` 只能作为辅助空挡入口，不提供锁主控或硬件急停能力。
- LLM Agent 只能输出结构化高层控制事件，不能输出 PWM、GPIO、电调校准、解锁、关闭保护或修改 ESP32 安全阈值。
- LLM 控制事件必须经过 Android schema 校验、权限检查和限幅，再由 ESP32 二次限幅；解析失败、超时、网络错误、会话断开或权限冲突时保持当前安全状态或回空闲。
- 语音前进三档/四档只能作为固定档位目标，仍必须受声控功率限制和 ESP32 二次限幅约束；声控功率默认 `70%`，可配置到最高 `100%`，提高前必须按岸上限流和低功率水下测试逐级验证。
- 云端实时语音下行播放时必须启用回声消除或耳机优先策略；Agent 回复音频不得被重新上传后触发下一条控制事件，用户打断时必须取消当前下行回复。
- `快点` / `慢点` 只能按设置页微调步进调整当前目标，不能绕过手动解锁、声控功率限制或 ESP32 侧限幅。
- App 只负责描述用户或上层导航“要做什么”，主控负责“具体怎么做”。App 不得实现推进转向内环，不得根据航向误差在本地计算最终左右推进器差速；最终 `L/R` 必须来自主控回传。
- 语音角度转向必须以 `MODE=TURN;DIR=...;ANGLE=...` 这类意图指令下发给主控，由主控读取当前航向、生成目标并执行转向闭环；未锁航时主控 IMU 失效、角度控制超时或控制心跳丢失必须取消并回空挡。
- 航向锁定中语音角度转向由主控移动锁航目标，普通油门、档位和微调只能更新基础油门，不能取消航向锁定。
- 航向锁定快速闭环必须运行在 ESP32 固件中，依赖主控 IMU 航向和角速度正常读取；主控 IMU 失效、控制心跳丢失、App 本地锁定、硬件急停、界面取消锁航或语音取消航向锁定时必须取消。
- 航向来源可在设置页选择主控 IMU 或手机指南针。手机模式要求手机固定在桨板上、相对船体方向不变；手机拿在手里、放入口袋或被乘员转动时不得用于锁航、角度转向或自动控制。手机航向超过 `1.5s` 无有效更新时，App 不得用手机航向生成新的自动控制目标，ESP32 也不得继续按旧 `PHDG` 放大差速。
- 主控 IMU 模式要求 `YBIMU=1` 且主控标准船头航向 `YBHDG`、角速度 `YBGZ` 最近有效。当前 IMU 安装为 `+X` 朝船头，固件把模块欧拉角 `YBY + 180°` 归一化为 `YBHDG`；Android 导航、控制、日志和锁航入口统一读取 `YBHDG`，不得在 App 端重新选择 `YBY` 算法或另加偏置。手机模式由 Android 把手机罗盘读数加船头偏置后作为 `PHDG` 上报，ESP32 用连续 `PHDG` 差分估计角速度。
- Android 下发固件锁航前必须确认当前选择的航向源有效：主控 IMU 模式看 `HSRC=YBIMU/YBINIT/YBAGE/YBHDG`；手机模式看本机传感器最近 `1.5s` 内有读数，并随控制命令发送 `H_SRC=PHONE;PHDG=...`。如果当前航向源为 `NONE` 或手机航向不可用，只能提示等待或拒绝进入锁航，不能先进入再被动退出。
- 固件锁航或任何涉及转向的模式激活时，Android 不得本地计算最终 `L/R`；Android 只能下发航向源、当前手机航向、目标航向、基础推力、限幅参数、角度转向意图和心跳，ESP32 回传的 `TURN/HLOCK/HERR/HPHASE/HCORR/HBOOST/L/R` 作为转向状态和输出权威来源。
- Android 不得在用户不知情时自动进入固件锁航；固件锁航必须由用户操作或明确的控制事件触发，并要求当前选择的航向源有效。自动导航路线规划仍在 Android，快速航向稳定内环使用 ESP32 固件统一执行。
- ICM20948 融合航向中期方案上线初期只能作为影子航向源；未完成静止、倾斜、电机低功率和水下低速对比测试前，不得作为自动导航控制来源。
- ESP32 融合航向质量不为 `OK`、I2C 读失败、磁场异常、陀螺饱和或航向超时时，Android 不得使用 ESP32 融合航向控制。
- 航向锁定保留可配置角度容差、最大转向角度和空档动态原地掉头反推上限，避免指南针抖动导致持续修正或空档原地修正过猛；当前高响应测试版本非空档超过容差后的最小差速修正为 `3%`，锁航/转向最终语义输出正推最高 `70%`、反推最高 `60%`，空档原地掉头只有在误差超过“容差 + 1°”安静保持带后才把小修正抬到约 `10%` 起转门槛，避免 `1°-2°` 小误差时在 `0%` 和有效推力之间来回跳。提高空档反推上限前只能按岸上限流、低功率、短时下水的顺序逐步验证。
- 锁航转向差速方向必须按当前推进器语义验证：目标在左侧、航向误差为负时，语义输出应右推进增加、左推进减少；目标在右侧、航向误差为正时，语义输出应左推进增加、右推进减少。该规则只属于主控闭环到用户语义 `L/R` 的映射，不得在 App 或 ESC 最终 PWM 反向配置中重复处理。
- 非空档航向锁定在基础推力绝对值低于 `70%` 时允许转向修正反推慢侧；基础推力达到或超过 `70%` 时不得主动反推，慢侧碰到 `0%` 后应牺牲差速，避免高推力下突然反向。
- 航向锁定短时补偿只在当前航向源有效、控制周期正常、输出未饱和时生效，并且只在当前锁航/转向会话内存在，不得写入 NVS 或跨水域/风向记忆。`HBOOST` 可由持续角速度误差积分得到，用来把“有误差但转动量不足”的差速逐步推过有效死区；主控 IMU 模式优先用 `YBGZ`，手机模式用 `PHDG` 差分角速度。误差超过安静保持带且角速度持续跟不上时，允许按受限最低爬升速度加大补偿，避免几十秒才到有效推力。`HFHAT` 这类扰动观测/模型学习必须等有效差速高于死区后才更新，避免把无效小油门误记成水流或电机特性。进入预测刹车、大斜率切换或角加速度尖峰时必须冻结或慢衰减短时补偿，避免瞬时门控把已积累差速直接打回起转门槛；传感异常、误差收敛或退出锁航时必须快速衰减/清零。`HWARN=HEADING_LOCK_DIVERGED` 表示方向、左右电机映射或航向源可能不可信；只有误差持续扩大且船头仍在远离目标方向转动时，才临时冻结 `HBOOST` 和转向修正，让输出回到基础推力；船头停止远离目标或开始回目标时必须清零 `HBOOST` 并自动恢复差速修正。补偿值必须通过 `HBOOST/HFHAT` 或等效字段回传，方便现场判断是否正在加力。
- 航向锁定提前反向力只能基于有效航向角速度或航向变化率；反打强度和保持时间必须随转动速度动态调整，并限制持续时间和最大反打；角速度变号或降到停稳阈值时必须立即退出反打，基础推力达到或超过 `70%` 时不得主动反推。加速度计只作为扰动/运动强度辅助判断，不直接决定 yaw 反打方向。
- 航向锁定误差如果持续同向扩大，且船头角速度仍在远离目标方向，ESP32 固件必须认为闭环方向、左右电机方向或航向源可能不可信，并回传 `HWARN=HEADING_LOCK_DIVERGED`；进入该告警后必须停止继续放大差速，冻结短时补偿和转向修正，维持基础推力输出。船头停止远离目标或开始回目标后必须自动退出该告警并恢复差速修正，避免大误差状态永久卡在等推力。但按用户授权规则，不得自动取消航向锁定或回传无授权 `HLOCK=OFF`。只有用户手动取消、用户语音取消、蓝牙断开，或明确硬安全故障才允许退出锁航。
- 左/右 ESC 反向只能作为 ESP32 主控 NVS 中的安装/出厂配置，通过 `ESC_CFG` 写入，并只在固件最终 PWM 映射层应用。App 只能在安装/维护设置中写入该配置；App 控制算法、ESP32 闭环算法、锁航、角度转向、自动导航、状态回传和控制日志中的 `L/R` 均为用户语义输出；不得读取、携带或暴露底层反向后的 raw 输出。
- 自动导航不得让 ESP32 独立执行未审查路线；第一版路线保存、航点推进和上层目标航向仍由 Android App 负责。Android 只把自动导航结果表达为目标航向、基础推进和心跳，由 ESP32 固件统一转向内环计算最终左右功率；不得保留 App 端自动导航差速转向算法。
- 自动导航执行前必须检查 GPS 已定位、卫星数满足最低要求、当前选择的航向源可用、路线至少 2 个点；任一条件失效必须停止自动导航并回空闲。
- 自动导航界面必须始终显示停止按钮；停止按钮必须立即发送空闲或锁定命令，不能只退出界面。
- `停止声控` 后 Android 端不得执行除 `开始声控` 之外的语音命令。
- 看门狗开启，重启后仍然保持未解锁。

## 岸上测试

- 先使用限流电源或串联保险测试。
- 单电机分别测试方向和油门范围。
- 双电机低油门联动测试。
- 记录电池线、ESC、接头、电机温度。

## 水下测试

- 静水低功率短时测试。
- 每次测试后检查进水、胶封、接头和温升。
- 逐步增加油门和持续时间。
- 保留测试日志，记录电压、电流、温度、故障和环境。
