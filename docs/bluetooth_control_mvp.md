# 经典蓝牙控制 MVP

## 目标

第一版 Android 控制端通过经典蓝牙 SPP 连接 ESP32，先跑通“配对、连接、发送手动控制命令、失联回空闲”的最小闭环。

## 蓝牙参数

| 项目 | 当前约定 |
| --- | --- |
| ESP32 蓝牙名称 | `SmartSUP-<三位编号>`，例如 `SmartSUP-001` |
| 蓝牙类型 | 经典蓝牙 SPP / RFCOMM |
| SPP UUID | `00001101-0000-1000-8000-00805F9B34FB` |
| 手机端 | Android 原生 App，在 App 设置页扫描名称前缀为 `SmartSUP-` 的设备并连接 |

Android App 会过滤设备名称前缀，当前固定为 `SmartSUP-`。如果目标 ESP32 尚未配对，连接时由系统弹出配对确认，不要求用户先切到系统蓝牙设置页。

## 设备编号

每个 ESP32 使用一个三位十进制编号，范围 `000-999`。编号在出厂第一次 USB 刷入时写入 ESP32 NVS/flash，用于生成经典蓝牙名称：

```text
SmartSUP-000
SmartSUP-001
SmartSUP-002
...
```

ESP32 固件本身不能知道“全项目下一个编号”是多少，因此编号分配由项目台账负责。台账文件是 `config/esp32_unit_registry.json`，出厂刷入流程见 [ESP32 出厂编号刷入流程](esp32_factory_provisioning.md)。

编号可以通过 USB 串口或已连接蓝牙 SPP 查询：

```text
ID?
```

响应格式：

```text
ID;VALUE=001;BT=SmartSUP-001;SRC=NVS;PROVISIONED=1
```

普通 App 不提供编号写入入口。`ID_SET` 不是用户命令，固件会返回 `ID;ERR=FACTORY_ONLY`。普通 Release/OTA 固件不会覆盖 NVS 中已有的编号。

## 命令格式

手机端按行发送 ASCII 文本，每行以 `\n` 结束：

```text
ARM=1;L=30;R=-20
SRC=VOICE;ARM=1;L=30;R=30;VMAX=70
SRC=VOICE;ARM=1;MODE=TURN;DIR=LEFT;ANGLE=30;TID=12;LREV=0;RREV=0;VMAX=70
SRC=APP;ARM=1;MODE=HEADING_LOCK;HLOCK=1;BASE=20;HTOL=4;HFULL=6;HREV=70;HID=7;LREV=0;RREV=0
SRC=VOICE;ARM=1;MODE=HEADING_LOCK;HLOCK=1;BASE=20;HTOL=4;HFULL=6;HREV=70;HID=8;HOFF=-10;LREV=0;RREV=0;VMAX=70
SRC=APP;ARM=1;L=20;R=20;H_SRC=PHONE;PHDG=123.4
```

字段说明：

| 字段 | 范围 | 说明 |
| --- | --- | --- |
| `SRC` | `APP` 或 `VOICE` | 可选字段；省略时按普通 App 控制处理，`VOICE` 表示语音控制输入 |
| `ARM` | `0` 或 `1` | `1` 表示 App 明确解锁，`0` 表示锁定 |
| `L` | `-100..100` | 左侧 ESC 有符号油门百分比，正值前进，负值后退 |
| `R` | `-100..100` | 右侧 ESC 有符号油门百分比，正值前进，负值后退 |
| `MODE` | `THROTTLE`、`TURN` 或 `HEADING_LOCK` | 可选字段；省略时按普通油门命令处理 |
| `DIR` | `LEFT` 或 `RIGHT` | `MODE=TURN` 时的目标转向方向 |
| `ANGLE` | `1..90` | `MODE=TURN` 时的相对目标角度，单位度 |
| `TID` | `1..65535` | 角度转向请求编号；同一编号重复发送只作为心跳，不重置目标 |
| `HLOCK` | `0` 或 `1` | `MODE=HEADING_LOCK` 时，`1` 表示开启航向锁定，`0` 表示取消 |
| `BASE` | `-100..100` | `MODE=HEADING_LOCK` 时的基础推进油门；固件会再次限幅 |
| `HTOL` | `1..20` | `MODE=HEADING_LOCK` 时的航向误差容差，单位度；省略时默认 `4°` |
| `HFULL` | `5..180` | `MODE=HEADING_LOCK` 时达到最大差速修正的误差角，单位度；必须大于 `HTOL`，省略时默认 `6°` |
| `HREV` | `0..100` | `MODE=HEADING_LOCK` 且 `BASE=0` 时允许的最大反推百分比；省略时默认 `70%` |
| `HOFF` | `-90..90` | 可选一次性字段；`MODE=HEADING_LOCK` 且新 `HID` 时，把锁航目标设置为当前航向加相对偏移，负值为左转。Android 首次发送后必须从心跳命令中移除 |
| `VMAX` | `5..100` | `SRC=VOICE` 时的声控功率限制，正反向使用同一绝对值上限；省略时默认 `70%` |
| `HID` | `1..65535` | 航向锁定请求编号；同一编号重复发送只作为心跳，不重置目标航向 |
| `LREV` / `RREV` | `0` 或 `1` | `MODE=TURN` 或 `MODE=HEADING_LOCK` 时由 App 下发的左右 ESC 方向反转设置 |
| `H_SRC` | `IMU` 或 `PHONE` | 当前航向来源；默认 `IMU`，设置页开启手机指南针后由 App 每个控制心跳发送 `PHONE` |
| `PHDG` | `-180..360` | `H_SRC=PHONE` 时的手机指南针航向，单位度；固件内部归一化到 `0..360` |

当前 MVP 按双向 ESC 处理，`0%` 对应 ESC 中位/空闲脉宽，`100%` 对应最大前进脉宽，`-100%` 对应最大后退脉宽。App 默认有油门限幅，初始为 30%，该限幅作为正反向的绝对值上限。

`MODE=TURN` 是一次性目标命令，不是普通油门命令。Android 会在语音识别到“左转 30 度”一类指令时生成新的 `TID`；ESP32 收到新 `TID` 后记录当前可用航向，计算目标相对角度，并在后续控制周期中用低速差速输出逼近目标。Android 心跳重复发送同一个 `TID` 时，ESP32 只刷新通信超时计时，不重新计算目标角度。

`MODE=HEADING_LOCK` 是持续航向保持命令。Android 控制页开启航向锁定或语音识别到“保持航向”时生成新的 `HID`；ESP32 收到新 `HID` 后把当前可用航向记录为目标航向。后续 Android 每 250ms 重发同一 `HID` 作为心跳，ESP32 不重新取目标，只按当前航向误差持续做低速差速修正。取消航向锁定、手动油门变化、锁定、急停、蓝牙失联、当前航向来源失效或 IMU 异常都会退出该模式。

航向锁定已经激活时，普通档位、油门微调和语音前进/后退只更新 `BASE`，不重新采样目标航向，也不退出锁航。语音角度转向不再发送 `MODE=TURN`，而是发送新的 `MODE=HEADING_LOCK` 命令和新 `HID`，并通过 `HOFF` 把目标航向相对当前航向偏移；例如锁航中“左转 10 度”首次发送 `HOFF=-10`。`HOFF` 只能作为一次性偏移字段发送，Android 发送成功后必须把后续心跳改成同一 `HID`、不带 `HOFF` 的保持命令；ESP32 也会记录已消费的 `HOFF` 请求编号，避免重复包再次叠加。除界面锁航按钮、语音“取消航向锁定”、急停/锁定、蓝牙失联、IMU 异常等安全状态外，正常控制命令不得退出航向锁定。

航向锁定初始控制策略：

- 目标航向来自开启瞬间的当前 IMU 磁力计航向；当前版本不再使用陀螺仪 Z 轴积分作为 `HDG`，避免静止漂移导致当前航向持续累加。磁力计航向会做轻度环形低通滤波，以压低静止时 1-3° 的抖动。
- 设置页可把航向来源切到手机指南针测试模式。该模式要求手机固定在桨板上且相对船体方向不变；手机拿在手里或转动手机时不能用于航向锁定。App 使用 Android rotation vector / geomagnetic rotation vector 读数，并随控制心跳发送 `H_SRC=PHONE;PHDG=...`。ESP32 超过 `1s` 未收到新的手机航向时，必须退出角度转向/航向锁定并回空挡锁定。
- 航向误差容差默认 `4°`，可在 Android 设置页调整。误差在容差内不主动修正，避免 IMU 漂移和噪声导致电机抖动。
- 误差超过容差后，按误差大小线性增加差速修正；默认 `6°` 及以上达到最大 `70%` 修正，该角度也可在 Android 设置页按 `1°` 步进调整。为避免空挡锁航时电机只给极小输出，当前高响应测试版本在超过容差后保留约 `10%` 最小差速修正。
- 默认采用“基础推进 + 差速修正”：有前进基础油门时优先一侧加推、一侧降推，不主动反推；基础油门为 `0%` 时允许双向原地修正，但反推侧受 `HREV` 限制，默认最多 `-70%`，可在 Android 设置页调整。
- 当前航向锁定按高响应测试处理：手动 App 开启航向锁定时保留当前档位/手动油门作为 `BASE`，不按语音限幅降档；语音开启航向锁定时基础油门和单侧输出均受 `VMAX` 限制，默认最高 `70%`，可配置到最高 `100%`。最大差速修正为 `70%`，手动 App 来源的航向锁定单侧输出最高允许到 `100%`。该设置只适合固定桨叶防护、岸上限流和低风险水下短测逐步验证；如果实测过猛，应调低声控功率限制或增加“保守/标准/激进”策略。

## 语音控制来源

Android 本地语音控制方案见 [Android 本地语音控制方案](voice_control_plan.md)。语音命令必须带 `SRC=VOICE`，并在 Android 和 ESP32 两侧同时限幅。

初始固件级语音限幅：

| 项目 | 初始值 |
| --- | --- |
| 声控功率限制 | 默认 `70%`，Android 设置页可在 `5-100%` 调整，正反向使用同一绝对值上限 |
| 语音左右差速最大差值 | `20%` |
| 语音单次角度转向最大角度 | `90°` |
| 角度转向超时 | `8s` |
| 航向锁定容差 | 默认 `4°`，App 设置页可配置 |
| 航向锁定最大误差参考 | 默认 `6°`，App 设置页可配置 |
| 航向锁定最大差速修正 | `70%` |
| 空档锁航最大反推 | 默认 `70%`，App 设置页可配置 |
| 语音航向锁定单侧输出最大值 | 跟随 `VMAX`，默认最高 `70%`，可配置到最高 `100%` |
| App 航向锁定单侧输出最大值 | `100%` |

语音命令不能把未解锁系统切换到解锁状态；只能在系统已解锁后改变低速目标，或在任何状态下执行 `ARM=0;L=0;R=0` 停止/锁定。高油门仍必须通过手动 App 控制或有线控制器，并受对应模式的限幅和安全保护约束。

角度转向依赖 ICM20948 内置 AK09916 磁力计 X/Y 计算当前航向。IMU 或磁力计未检测到、读取失败、通信超时或系统未解锁时，ESP32 必须拒绝或取消角度转向，并回到空闲/保守输出。磁力计航向容易受电机、电池、大电流线和金属件干扰，正式下水前必须在实际安装位置做低功率验证。

## 磁力计校准命令

Android 设置页提供航向校准入口，通过经典蓝牙 SPP 发送原始命令：

```text
MAG_CAL;ACTION=START
MAG_CAL;ACTION=SAVE
MAG_CAL;ACTION=CLEAR
MAG_CAL;ACTION=STATUS
```

`START`、`SAVE` 和 `CLEAR` 都是安全敏感命令。ESP32 收到后必须立即锁定、左右油门回空挡、取消角度转向和航向锁定；语音或普通油门命令不能绕过这个行为。`START` 后固件在每次 AK09916 新数据到达时记录 X/Y 最小最大值；操作时应在实际安装位置缓慢改变板体方向，覆盖尽量完整的水平圆周。`SAVE` 至少要求 `80` 个有效样本，且 X/Y 覆盖范围都不小于 `100` 个原始计数；保存后的硬铁偏移和简单轴向缩放写入 ESP32 NVS，重启后继续生效。`CLEAR` 删除 NVS 里的磁力计校准参数。

校准命令会返回 `STATUS` 字段：

| 字段 | 说明 |
| --- | --- |
| `MCAL` | `ACTIVE` 表示正在采样，`SAVED` 表示已有保存校准，`NONE` 表示无保存校准 |
| `MCNT` | 当前校准会话采样数量 |
| `MRX` / `MRY` | 当前校准会话 X/Y 原始覆盖范围 |
| `MOX` / `MOY` | 已保存或当前使用的 X/Y 硬铁偏移 |
| `MSX` / `MSY` | 已保存或当前使用的 X/Y 轴向缩放 |

## 状态回包

ESP32 每秒通过蓝牙 SPP 发送一行 `STATUS` 状态。Android App 解析 `IMU`、`HDG`、GPS 字段和角度转向时的 `TARGET` 字段，并在控制页显示 IMU 状态、航向角、GPS 模块状态、定位状态、卫星数量和经纬度：

```text
STATUS;ARMED=0;L=0;R=0;CMD_SRC=APP;MODE=THROTTLE;LPWM=1500;RPWM=1500;IMU=1;HDG=12.3;GPS=1;PPS=0;GPS_SENT=32;GPS_BAUD=115200;GPS_FIX=0;GPS_SAT=0;GPS_ANT=OPEN;ID=001;BT=SmartSUP-001;ID_SRC=NVS
STATUS;ARMED=1;L=8;R=18;CMD_SRC=VOICE;MODE=TURN;LPWM=1540;RPWM=1590;IMU=1;HDG=12.3;GPS=1;PPS=1;GPS_SENT=120;GPS_BAUD=115200;GPS_FIX=1;GPS_SAT=8;GPS_ANT=OK;GPS_LAT=22.123456;GPS_LON=113.123456;TURN=ACTIVE;TID=12;TARGET=42.3;TERR=30.0;ID=001;BT=SmartSUP-001;ID_SRC=NVS
STATUS;ARMED=1;L=24;R=16;CMD_SRC=APP;MODE=HEADING_LOCK;LPWM=1620;RPWM=1580;IMU=1;HDG=10.2;GPS=1;PPS=1;GPS_SENT=120;GPS_BAUD=115200;GPS_FIX=1;GPS_SAT=8;GPS_ANT=OK;GPS_LAT=22.123456;GPS_LON=113.123456;HLOCK=ACTIVE;HID=7;TARGET=0.0;HERR=-10.2;HCORR=-70;HTOL=4;HFULL=6;HREV=70;ID=001;BT=SmartSUP-001;ID_SRC=NVS
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `IMU` | `1` 表示 IMU 当前可用，`0` 表示 IMU 不可用或读取失败 |
| `HDG` | ESP32 基于 ICM20948 内置 AK09916 磁力计 X/Y 直接计算的当前航向角，单位度 |
| `HSRC` | `PHONE` 表示当前航向来自手机指南针，`MAG` 表示当前航向来自 ESP32 磁力计，`NONE` 表示无航向来源 |
| `PHDG` | 手机指南针最近一次航向，单位度；仅 `H_SRC=PHONE` 时回传 |
| `PHDG_AGE` | 手机指南针航向距当前的毫秒数；超过 `1000ms` 不能用于锁航 |
| `MCAL` | 磁力计校准状态：`ACTIVE`、`SAVED` 或 `NONE` |
| `MCNT` | 当前校准会话采样数量 |
| `TARGET` | `TURN=ACTIVE` 或 `HLOCK=ACTIVE` 时的目标航向角，单位度 |
| `HLOCK` | `ACTIVE` 表示航向锁定闭环正在执行 |
| `HID` | 当前航向锁定请求编号 |
| `HERR` | 航向锁定误差，单位度 |
| `HCORR` | 航向锁定差速修正百分比，正值表示左侧增加、右侧降低 |
| `HTOL` | 当前航向锁定容差，单位度 |
| `HFULL` | 当前航向锁定达到最大差速修正的误差角，单位度 |
| `HREV` | 当前空档锁航最大反推百分比 |
| `GPS` | `1` 表示 GPS 串口最近有数据，`0` 表示最近无 GPS 串口数据 |
| `PPS` | `1` 表示最近检测到 PPS 秒脉冲，`0` 表示未检测到 |
| `GPS_BAUD` | 当前锁定或正在扫描的 GPS 串口波特率 |
| `GPS_FIX` | `1` 表示 NMEA 报告已定位，`0` 表示未定位 |
| `GPS_SAT` | NMEA `GGA` 语句报告的卫星数量 |
| `GPS_ANT` | GPS 天线状态，`OK`、`OPEN` 或 `UNKNOWN` |
| `GPS_LAT` / `GPS_LON` | 定位有效时的纬度/经度，十进制度 |

`HDG` 当前来自磁力计 X/Y 平面航向，不再来自陀螺积分；静止时不应持续累加或累减。固件会按最短角度差做轻度低通滤波，避免显示和锁航误差被瞬时噪声带动。由于未做完整硬/软铁校准和倾斜补偿，电机、电池、电流线、螺丝和板体倾斜都可能造成角度偏差，当前主要用于低速短时验证。

## GPS 轨迹同步

GPS 轨迹记录和回放方案见 [GPS 实时定位、轨迹记录和回放方案](gps_track_plan.md)。第一版约定：

- ESP32 以 `1Hz` 记录有效 GPS 定位点。
- 每个轨迹点只保存 UTC 时间、纬度和经度；内部 `session_id` 和 `seq` 只用于同步和环形覆盖。
- 未定位、时间无效、经纬度无效、卫星数不足或推算速度超过 `5 m/s` 的点不写入。
- 轨迹分区存满后自动覆盖最旧点。
- Android 通过经典蓝牙 SPP 增量同步，并在手机本地长期保存。

同步命令：

```text
LOG_INFO
LOG_READ;FROM=1001;LIMIT=32
```

响应：

```text
LOG_INFO;REC=16;CAP=61440;COUNT=1200;OLDEST=1;NEWEST=1200;SESSION=12;RATE=1
LOG_BEGIN;FROM=1001;COUNT=32
LOG_POINT;SEQ=1001;SID=12;T=1781510400;LAT=221234567;LON=1131234567
LOG_END;NEXT=1033
```

轨迹同步命令不得改变 `ARM`、油门、航向锁定和失联保护状态。同步时 Android 仍需持续发送控制心跳；ESP32 仍按控制命令超时进入锁定状态。

## 手动控制 UI 约定

- 控制页左右两侧为竖向双向油门，滑块中位为 `0%`。
- 手指向上拖动为前进，向下拖动为后退。
- 松开任意油门滑块后，该侧油门自动回到当前档位值。
- 中间档位为单行矩形条，矩形内不显示文字，使用颜色深浅体现速度，并用小三角指示当前档位。

- 航向锁定闭环激活时，左右滑动条显示 ESP32 `STATUS` 回包里的实际 `L/R` 输出；手动拖动滑动条仍会取消航向锁定并回到手动控制。

| 档位 | 油门值 | 说明 |

| 空挡 | `0%` | 默认安全档 |
| 前进 1 | `20%` | 低速前进档 |
| 前进 2 | `40%` | 中速前进档 |
| 前进 3 | `60%` | 较高前进档 |
| 前进 4 | `80%` | 高速前进档 |

点击档位会设置当前档位，并把左右油门回到该档位值；如果设置页限幅低于档位值，则按限幅夹取。航向锁定已激活时，点击档位不取消航向锁定，只更新 `BASE` 基础推进油门并沿用同一个 `HID`，目标航向不重新采样。手动拖动油门只更新锁航基础油门；急停、锁定、失联超时会退出航向锁定并回到安全状态。

控制页提供当前目标油门的微调加/减按钮，默认每次调整 `3%`，可在 App 设置页改为 `1-10%`。微调只改变当前临时目标，不改写设置页里的档位百分比；选择新档位、锁定、急停、断线或采样模式会清零微调。前进时“加”提高正向油门，后退时“加”提高倒退绝对值，“减”向 `0%` 收敛。所有微调结果仍受 App 正反向油门限幅、声控限幅和 ESP32 侧最终限幅约束。

## ESC 方向反转

- App 设置页提供左/右 ESC 独立方向反转开关，用于电机三相线接线方向不确定时的软件校正。
- 方向反转在 App 发送命令前生效：例如左 ESC 开启反转后，控制页 `L=30` 会实际发送为 `L=-30`。
- `0%` 空挡不受反转影响。
- 切换任一方向反转开关时，App 会立即锁定并把左右油门回空挡，避免带油门状态下瞬间反向。

## 心跳和失联保护

- Android 连接后每 250ms 重发当前控制命令。
- ESP32 只有收到格式正确的命令后才更新目标油门。
- ESP32 超过 1000ms 没收到有效命令时，立即进入锁定状态，目标油门回到空闲。
- ESP32 上电默认锁定；蓝牙连接成功不等于解锁。
- App 急停或锁定时发送 `ARM=0;L=0;R=0`。
