# 经典蓝牙控制 MVP

> 历史方案：第一版实时遥控链路已改为 [ESP-NOW 遥控 MVP](espnow_control_mvp.md)。本文件保留 Android 经典蓝牙 SPP 方案和旧协议细节，后续只作为维护链路或兼容参考。

## 目标

第一版 Android 控制端通过经典蓝牙 SPP 连接 ESP32，先跑通“配对、连接、发送手动控制命令、失联回空闲”的最小闭环。

## 蓝牙参数

| 项目 | 当前约定 |
| --- | --- |
| ESP32 蓝牙名称 | `SmartSUP-<三位编号>`，例如 `SmartSUP-001` |
| 蓝牙类型 | 经典蓝牙 SPP / RFCOMM |
| SPP UUID | `00001101-0000-1000-8000-00805F9B34FB` |
| 手机端 | Android 原生 App，在 App 设置页扫描名称前缀为 `SmartSUP-` 的设备并连接 |

Android App 会过滤设备名称前缀，当前固定为 `SmartSUP-`。如果目标 ESP32 尚未配对，连接时由系统弹出配对确认，不要求用户先切到系统蓝牙设置页。

## 设备编号

每个 ESP32 使用一个三位十进制编号，范围 `000-999`。编号在出厂第一次 USB 刷入时写入 ESP32 NVS/flash，用于生成经典蓝牙名称：

```text
SmartSUP-000
SmartSUP-001
SmartSUP-002
...
```

ESP32 固件本身不能知道“全项目下一个编号”是多少，因此编号分配由项目台账负责。台账文件是 `config/esp32_unit_registry.json`，出厂刷入流程见 [ESP32 出厂编号刷入流程](esp32_factory_provisioning.md)。

编号可以通过 USB 串口或已连接蓝牙 SPP 查询：

```text
ID?
```

响应格式：

```text
ID;VALUE=001;BT=SmartSUP-001;SRC=NVS;PROVISIONED=1
```

普通 App 不提供编号写入入口。`ID_SET` 不是用户命令，固件会返回 `ID;ERR=FACTORY_ONLY`。普通 Release/OTA 固件不会覆盖 NVS 中已有的编号。

## 命令格式

手机端按行发送 ASCII 文本，每行以 `\n` 结束：

```text
ARM=1;L=30;R=-20
SRC=VOICE;ARM=1;L=30;R=30;VMAX=70
SRC=VOICE;ARM=1;L=14;R=-14;VMAX=70
SRC=APP;ARM=1;L=28;R=12
```

字段说明：

| 字段 | 范围 | 说明 |
| --- | --- | --- |
| `SRC` | `APP` 或 `VOICE` | 可选字段；省略时按普通 App 控制处理，`VOICE` 表示语音控制输入 |
| `ARM` | `0` 或 `1` | `1` 表示 App 明确解锁，`0` 表示锁定 |
| `L` | `-100..100` | 左侧 ESC 有符号油门百分比，正值前进，负值后退 |
| `R` | `-100..100` | 右侧 ESC 有符号油门百分比，正值前进，负值后退 |
| `MODE` | `THROTTLE` | 当前版本只允许普通油门命令；角度转向和航向锁定由 Android App 本地计算后下发最终 `L/R` |
| `VMAX` | `5..100` | `SRC=VOICE` 时的声控功率限制，正反向使用同一绝对值上限；省略时默认 `70%` |
| `H_SRC` / `PHDG` | 停用 | 当前航向不再发给 ESP32；手机指南针只在 Android App 本地使用 |

当前 MVP 按双向 ESC 处理，`0%` 对应 ESC 中位/空闲脉宽，`100%` 对应最大前进脉宽，`-100%` 对应最大后退脉宽。App 默认有油门限幅，初始为 30%，该限幅作为正反向的绝对值上限。

2026-06-20 岸上低功率实测显示当前电机/ESC 组合约 `9%` 才开始缓慢转动。ESP32 在最终 PWM 映射前应用最低有效推进死区：`-8%..8%` 下发到 ESC 时保持中位/空闲脉宽，`9%` 及以上或 `-9%` 及以下才进入线性 PWM 映射。Android 仍可以显示原始目标百分比；判断实际 ESC 输出时以 `STATUS` 中的 `LPWM/RPWM` 为准。

`MODE=TURN` 停用。Android 会在语音识别到“左转 30 度”一类指令时用手机指南针在 App 本地计算目标航向和差速输出，心跳仍只发送普通 `ARM/L/R` 功率命令。

`MODE=HEADING_LOCK` 停用。Android 控制页开启航向锁定或语音识别到“保持航向”时，App 按设置页选择的航向源记录目标航向；默认来源是手机指南针，测试开关可切到 ESP32 新九轴 IMU 的可选安装算法航向。后续每 `100ms` 在 App 本地计算误差和左右 ESC 功率，发送普通 `ARM/L/R` 心跳。ESP32 不保存锁航状态，也不参与航向计算。

航向锁定已经激活时，普通档位、油门微调和语音前进/后退只更新 App 本地基础油门，不重新采样目标航向，也不退出锁航。手动拖动单侧 ESC 滑条时，只更新该侧基础油门，另一侧基础油门保持不变；航向修正继续叠加在两侧基础油门上。语音角度转向不再发送旧 `MODE` 字段，而是在 App 内把本地目标航向按手机当前航向加相对偏移更新；例如锁航中“左转 10 度”会把 App 本地目标设为当前手机航向减 `10°`。除界面锁航按钮、语音“取消航向锁定”、急停/锁定、蓝牙失联、手机航向失效等安全状态外，正常控制命令不得退出航向锁定。

航向锁定初始控制策略：

- 目标航向来自开启瞬间的 Android 设置页所选航向源。默认使用手机指南针；“航向锁定使用 ESP32 IMU”只作为测试开关，使用新九轴 IMU 模块内置算法输出的欧拉角 `YBY`。设置页只保留 `YBY 原始` 和 `YBY 反向` 两种方向选择；桨板船头对准正北时可点击“设为正北 0°”保存零点偏置，也可用“对齐手机航向”重新校准 `offset`。切换来源、切换算法或校准时会取消当前锁航并回空挡。
- 手机必须固定在桨板上且相对船体方向不变；手机拿在手里或转动手机时不能用于航向锁定。App 使用 Android rotation vector / geomagnetic rotation vector 读数，不再把手机航向发给 ESP32。
- IMU 测试模式要求 `YBIMU=1` 且 `YBY` 最近 `500ms` 内有效；IMU 航向超时、蓝牙失联、App 锁定、硬件急停或界面/语音取消锁航时必须退出锁航。
- 航向误差容差默认 `2°`，可在 Android 设置页调整。误差在容差内不主动修正，避免指南针噪声导致电机抖动。
- 误差超过容差后，按误差大小线性增加差速修正；默认 `6°` 及以上达到最大 `70%` 修正，该角度也可在 Android 设置页按 `1°` 步进调整。为匹配当前 ESC 实测启动门槛，超过容差后的最小差速修正为 `3%`。
- 持续锁航增加 3 秒误差窗口自适应补偿：如果最近约 `3s` 内航向误差始终同向、超过容差且没有明显变小，App 每个控制周期小步增加差速补偿，最高额外 `50%`；误差仍同向且超过容差但正在变小时保持当前补偿，变号或回到容差内时按更快速度衰减。该补偿只作用于持续锁航，不作用于一次性角度转向，且最终仍受单侧输出限幅约束。
- 默认采用“基础推进 + 差速修正”：当一侧输出超过当前来源允许上限时，App 会把左右输出整体下移，优先保留左右推力差，避免高档位撞到 `100%` 后反而减小转向差；有前进基础油门的一侧不主动反推，有后退基础油门的一侧不主动正推。
- 基础油门为 `0%` 时使用动态原地掉头差值：误差超过容差后，App 按转向方向发送一侧正推、一侧反推，左右总差值随航向误差从默认 `20%` 线性增加到 `60%`；`20%` 总差值约等于单侧 `10%`，用于避开当前电机起转死区。低速满修正角度至少按 `45°` 计算，设置页可调整“空档最小转向差/最大转向差”。误差回到容差内仍输出 `0/0`。
- Android 控制页在航向锁定和 AI 原地掉头时同时显示“用户视角输出”和“实际下发 L/R”。实际下发值会应用左右 ESC 反向设置；如果某侧开启反向，UI 看到的后退目标可能会转换成相反符号的 `L/R` 线控值，这是为了让物理推力方向仍符合用户视角。
- 当前航向锁定按高响应测试处理：手动 App 开启航向锁定时保留当前档位/手动油门作为 `BASE`，不按语音限幅降档；语音开启航向锁定时基础油门和单侧输出均受 `VMAX` 限制，默认最高 `70%`，可配置到最高 `100%`。非空档差速修正随当前基础推进动态调整：低速最大修正约 `25%`，并把满修正角度放宽到至少 `45°`；基础推进从 `10%` 到 `50%` 逐步过渡，`50%` 及以上使用设置页配置的最大误差参考和最高 `70%` 差速修正。手动 App 来源的航向锁定单侧输出最高允许到 `100%`。该设置只适合固定桨叶防护、岸上限流和低风险水下短测逐步验证；如果实测过猛，应调低声控功率限制或增加“保守/标准/激进”策略。

## 语音控制来源

Android 本地语音控制方案见 [Android 本地语音控制方案](voice_control_plan.md)。旧的固件级语音控制命令使用 `SRC=VOICE`，并在 Android 和 ESP32 两侧同时限幅。

2026-06-19 起，实时 AI 的普通档位、左右推进百分比、空挡、原地掉头和航向目标都改由 Android App 校验/计算后按普通 `SRC=APP;ARM=1;L=...;R=...` 心跳下发，避免同一个 `-40%` 或同一组转向 `L/R` 在手动路径和 AI 路径上经过不同的 ESP32 `SRC=VOICE` 限制逻辑。AI 仍不能解锁，且数值仍先受 App 当前手动油门上限或语音功率限制约束。

初始固件级语音限幅：

| 项目 | 初始值 |
| --- | --- |
| 声控功率限制 | 默认 `70%`，Android 设置页可在 `5-100%` 调整，正反向使用同一绝对值上限 |
| 语音左右差速最大差值 | `20%` |
| 语音单次角度转向最大角度 | `90°` |
| 角度转向超时 | `8s` |
| 航向锁定容差 | 默认 `2°`，App 设置页可配置 |
| 航向锁定最大误差参考 | 默认 `6°`，App 设置页可配置 |
| 航向锁定最大差速修正 | 非空档低速约 `25%`，`50%` 及以上推进最高 `70%` |
| 空档原地掉头动态差值 | 左右总差值默认 `20%-60%`，App 设置页可配置 |
| 语音航向锁定单侧输出最大值 | 跟随 `VMAX`，默认最高 `70%`，可配置到最高 `100%` |
| App 航向锁定单侧输出最大值 | `100%` |

语音命令不能把未解锁系统切换到解锁状态；只能在系统已解锁后改变低速目标。普通“停止/空挡”只把基础推进设为 `0%`，航向锁定中不得取消锁航；只有明确“急停/锁定/锁主控”或本地安全保护才发送 `ARM=0;L=0;R=0`。高油门仍必须通过手动 App 控制或有线控制器，并受对应模式的限幅和安全保护约束。

角度转向依赖 Android 设置页所选航向源。默认使用手机指南针；切到 ESP32 IMU 测试模式后使用设置页选定的 IMU 航向算法。所选航向不可用、通信超时或系统未解锁时，App 必须拒绝或取消角度转向，并回到空闲/保守输出。ESP32 不独立执行角度转向。

## 磁力计校准命令

磁力计航向方式已停用，Android 设置页不再提供磁力计校准入口。固件仍可保留维护命令用于历史兼容，但它不参与航向锁定、角度转向或当前航向显示。

## 状态回包

ESP32 常规状态下每秒通过蓝牙 SPP 发送一行 `STATUS` 状态；检测到新九轴 IMU 后，IMU 测试状态上报提高到约 `10Hz`。Android App 解析推进输出、GPS 和设备状态；当前锁航目标和误差由 App 本地计算，不再依赖 ESP32 `HDG/TARGET`：

```text
STATUS;ARMED=0;FW=v0.2.8;L=0;R=0;CMD_SRC=APP;MODE=THROTTLE;LPWM=1500;RPWM=1500;GPS=1;PPS=0;GPS_SENT=32;GPS_BAUD=115200;GPS_FIX=0;GPS_SAT=0;GPS_ANT=OPEN;ID=001;BT=SmartSUP-001;ID_SRC=NVS
STATUS;ARMED=1;FW=v0.2.8;L=24;R=16;CMD_SRC=APP;MODE=THROTTLE;LPWM=1620;RPWM=1580;GPS=1;PPS=1;GPS_SENT=120;GPS_BAUD=115200;GPS_FIX=1;GPS_SAT=8;GPS_ANT=OK;GPS_LAT=22.123456;GPS_LON=113.123456;GPS_SPD_KMH=3.6;ID=001;BT=SmartSUP-001;ID_SRC=NVS
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `FW` | ESP32 当前固件版本，用于 App 连接后显示和 OTA 后验证 |
| `GPS` | `1` 表示 GPS 串口最近有数据，`0` 表示最近无 GPS 串口数据 |
| `PPS` | `1` 表示最近检测到 PPS 秒脉冲，`0` 表示未检测到 |
| `GPS_BAUD` | 当前锁定或正在扫描的 GPS 串口波特率 |
| `GPS_FIX` | `1` 表示 NMEA 报告已定位，`0` 表示未定位 |
| `GPS_SAT` | NMEA `GGA` 语句报告的卫星数量 |
| `GPS_ANT` | GPS 天线状态，`OK`、`OPEN` 或 `UNKNOWN` |
| `GPS_LAT` / `GPS_LON` | 定位有效时的纬度/经度，十进制度 |
| `GPS_SPD_KMH` | 定位有效时的地速，单位 km/h，来自 NMEA `RMC` 语句 |

App 控制页的主航向显示仍来自手机指南针；控制页和设置页的“IMU 航向”显示为当前 `YBY 原始/反向` 加偏置后的船体航向，设置页 IMU 观测卡保留原始 `YBY`、当前偏置和手机差值用于排查。ESP32 固件如果回传历史 `IMU/HDG/MCAL` 字段，Android 只把它们作为维护信息。

ICM20948 融合航向中期方案的第一阶段会额外回传 IMU 观测字段，用于 Android 维护界面对比和日志分析，不参与自动导航或航向锁定：

| 字段 | 说明 |
| --- | --- |
| `IAX` / `IAY` / `IAZ` | ICM20948 加速度计三轴，单位 g，当前按默认 `±2g` 量程换算 |
| `IGX` / `IGY` / `IGZ` | ICM20948 陀螺仪三轴，单位 dps，当前按 `±250dps` 量程换算 |
| `IMX` / `IMY` / `IMZ` | AK09916 磁力计三轴原始计数 |
| `IMAG` | AK09916 原始三轴磁场模长，用于观察磁干扰和异常降权条件 |

新 Yahboom 九轴 IMU 模块通过 I2C 地址 `0x23` 输出板载融合数据，ESP32 只作为 telemetry 回传，Android 控制页和设置页显示为“ESP32 IMU”。默认情况下这些字段只做观测；设置页开启“航向锁定使用 ESP32 IMU”后，航向锁定、角度转向、地图箭头和航迹线锁定可使用当前选定的 IMU 算法：

当 `YBIMU=1` 时，ESP32 为 IMU 测试把蓝牙 `STATUS` 回包周期从常规 `1Hz` 提高到约 `10Hz`；未检测到新 IMU 时仍使用常规 `1Hz`。

| 字段 | 说明 |
| --- | --- |
| `YBIMU` | 新 Yahboom IMU 是否在线，`1` 在线，`0` 离线 |
| `YBAX` / `YBAY` / `YBAZ` | 三轴加速度，单位 `g` |
| `YBGZ` | Z 轴角速度，单位 `rad/s` |
| `YBQW` / `YBQX` / `YBQY` / `YBQZ` | 四元数，顺序 `w, x, y, z`；当前 Android 控制不使用，仅保留观测 |
| `YBR` / `YBP` / `YBY` | 欧拉角 roll、pitch、yaw，单位 `deg`；`YBY` 是当前 IMU 航向测试输入 |

## GPS 轨迹同步

GPS 轨迹记录和回放方案见 [GPS 实时定位、轨迹记录和回放方案](gps_track_plan.md)。第一版约定：

- ESP32 以 `1Hz` 记录有效 GPS 定位点。
- 每个轨迹点只保存 UTC 时间、纬度和经度；内部 `session_id` 和 `seq` 只用于同步和环形覆盖。
- 未定位、时间无效、经纬度无效、卫星数不足或推算速度超过 `5 m/s` 的点不写入。
- 轨迹分区存满后自动覆盖最旧点。
- Android 通过经典蓝牙 SPP 增量同步，并在手机本地长期保存。

同步命令：

```text
LOG_INFO
LOG_READ;FROM=1001;LIMIT=256
```

响应：

```text
LOG_INFO;REC=16;CAP=61440;COUNT=1200;OLDEST=1;NEWEST=1200;SESSION=12;RATE=1
LOG_BEGIN;FROM=1001;COUNT=32
LOG_POINT;SEQ=1001;SID=12;T=1781510400;LAT=221234567;LON=1131234567
LOG_END;NEXT=1033
```

轨迹同步命令不得改变 `ARM`、油门、航向锁定和失联保护状态。同步时 Android 仍需持续发送控制心跳；ESP32 仍按控制命令超时进入锁定状态。

## 手动控制 UI 约定

- 控制页左右两侧为竖向双向油门，滑块中位为 `0%`。
- 手指向上拖动为前进，向下拖动为后退。
- 松开任意油门滑块后，该侧油门自动回到当前档位值。
- 中间档位为单行矩形条，矩形内不显示文字，使用颜色深浅体现速度，并用小三角指示当前档位。
- 档位百分比在 App 设置页分别配置，当前默认值如下：
- 航向锁定闭环激活时，左右滑动条优先显示 App 本地计算出的实际 `L/R` 输出，避免等待 ESP32 `STATUS` 回包造成延迟；手动拖动单侧滑条只更新该侧锁航基础油门，不取消航向锁定，也不改变另一侧基础油门。

| 档位 | 油门值 | 说明 |
| --- | --- | --- |
| 后退 3 | `-60%` | 较高倒退档 |
| 后退 2 | `-40%` | 较高倒退档 |
| 后退 1 | `-20%` | 低速倒退档 |
| 空挡 | `0%` | 默认安全档 |
| 前进 1 | `20%` | 低速前进档 |
| 前进 2 | `40%` | 中速前进档 |
| 前进 3 | `60%` | 较高前进档 |
| 前进 4 | `80%` | 高速前进档 |

点击档位会设置当前档位，并把左右油门回到该档位值；如果设置页限幅低于档位值，则按限幅夹取。航向锁定已激活时，点击档位不取消航向锁定，只把左右 App 本地基础推进油门更新为该档位，目标航向不重新采样。手动拖动油门只更新被拖动一侧的锁航基础油门；急停、锁定、失联超时会退出航向锁定并回到安全状态。

控制页提供当前目标油门的微调加/减按钮，默认每次调整 `3%`，可在 App 设置页改为 `1-10%`。微调只改变当前临时目标，不改写设置页里的档位百分比；选择新档位、锁定、急停、断线或采样模式会清零微调。前进时“加”提高正向油门，后退时“加”提高倒退绝对值，“减”向 `0%` 收敛。所有微调结果仍受 App 正反向油门限幅、声控限幅和 ESP32 侧最终限幅约束。

## ESC 方向反转

- App 设置页提供左/右 ESC 独立方向反转开关，用于电机三相线接线方向不确定时的软件校正。
- 方向反转在 App 发送命令前生效：例如左 ESC 开启反转后，控制页 `L=30` 会实际发送为 `L=-30`。
- `0%` 空挡不受反转影响。
- 切换任一方向反转开关时，App 会立即锁定并把左右油门回空挡，避免带油门状态下瞬间反向。

## 心跳和失联保护

- Android 连接后每 `100ms` 重发当前控制命令。
- Android 进入后台、息屏或切到其他 App 时，已连接状态必须启动前台服务并持有 `PARTIAL_WAKE_LOCK`，以维持蓝牙控制心跳；用户主动断开蓝牙后释放服务和 wake lock。
- ESP32 只有收到格式正确的命令后才更新目标油门。
- ESP32 超过 1000ms 没收到有效命令时，立即进入锁定状态，目标油门回到空闲。
- ESP32 上电默认锁定；蓝牙连接成功不等于解锁。
- App 急停或锁定时发送 `ARM=0;L=0;R=0`。
