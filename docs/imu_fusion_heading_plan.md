# ICM20948 融合航向中期方案

## 目标

中期目标是在 ESP32 上把 ICM20948 的陀螺仪、加速度计和 AK09916 磁力计做成融合航向源，用来接近 Android 手机 `Rotation Vector` 的稳定性。第一阶段只作为“影子航向源”上报给 Android 对比，不参与航向锁定、角度转向或自动导航控制。

当前正式控制源仍然是 Android 手机指南针航向。手机必须固定在桨板上且相对船体方向不变。

## 当前问题

现有固件的 ESP32 航向主要来自 AK09916 磁力计 X/Y：

```text
heading = atan2(calibratedY, calibratedX)
```

这个方式有几个问题：

- 没有使用加速度计做倾斜补偿。
- 没有使用陀螺仪提供短时间稳定性。
- 磁力计校准只有 X/Y 平面硬铁偏移和简单缩放，不能覆盖完整 3D 软铁误差。
- 磁力计读数直接影响航向，静止时也会把噪声转成角度跳动。
- 当前 `HDG` 只是维护字段，不应参与控制闭环。

手机放在同样位置仍然稳定，说明当前主要短板在算法和标定，而不一定是安装位置单一问题。

## 方案边界

- ICM20948 融合航向上线初期只上报，不控制推进。
- Android App 显示手机航向和 ESP32 融合航向的差值，用于测试和日志。
- 没有完成稳定性测试前，不允许把 ESP32 融合航向设为自动导航或锁航默认来源。
- 手机航向失效时，不能自动无条件切到 ESP32 融合航向；必须另行设计人工确认和安全策略。

## 数据链路

ICM20948V2 模块提供：

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| 加速度计 `ax/ay/az` | ICM20948 | 估计重力方向、roll/pitch、倾斜补偿 |
| 陀螺仪 `gx/gy/gz` | ICM20948 | 短时间姿态积分，压低航向抖动 |
| 磁力计 `mx/my/mz` | AK09916 | 长时间 yaw 校正 |

推荐固件内部姿态状态：

```text
quaternion q
roll / pitch / yaw
gyro bias
mag calibration
heading quality
```

## 分阶段实现

### 阶段 1：观测数据补齐

固件读取并上报原始/标定后的 9 轴数据摘要：

- 加速度计三轴。
- 陀螺仪三轴。
- 磁力计三轴。
- 磁场强度模长。
- 当前裸磁力计航向。

目标是确认轴向、噪声、量程和采样周期是否正常。

当前第一步固件维护字段：

| 字段 | 说明 |
| --- | --- |
| `IAX` / `IAY` / `IAZ` | 加速度计三轴，单位 g |
| `IGX` / `IGY` / `IGZ` | 陀螺仪三轴，单位 dps |
| `IMX` / `IMY` / `IMZ` | 磁力计三轴原始计数 |
| `IMAG` | 磁力计三轴原始模长 |

### 阶段 2：倾斜补偿磁航向

用加速度计估计 roll/pitch，把磁力计投影到水平面后计算航向：

```text
roll/pitch = accel -> gravity direction
horizontal mag = rotate mag by roll/pitch
heading = atan2(horizontal_y, horizontal_x)
```

该阶段仍不使用陀螺仪融合，只验证倾斜补偿是否能减少摆放角度变化导致的航向偏差。

### 阶段 3：Mahony/Madgwick 姿态融合

实现或接入轻量 9 轴 AHRS：

- 陀螺仪负责短时间平滑。
- 加速度计约束 roll/pitch。
- 磁力计只慢速校正 yaw。
- 磁力计异常时降低权重或临时拒绝。

初始建议：

| 参数 | 初始建议 |
| --- | --- |
| 更新率 | `50Hz` |
| 融合算法 | Mahony 或 Madgwick |
| 陀螺零偏校准 | 上电静止采样 `2-5s` |
| 磁力计校准 | 3D 硬铁偏移 + 轴向缩放，先不做完整矩阵软铁 |
| 磁力计异常判定 | 磁场模长偏离最近稳定均值 `25-35%` 时降权 |
| 输出航向低通 | 环形低通，时间常数约 `0.3-0.8s` |

当前中期实现采用 Adafruit AHRS 库中的 Mahony 四元数融合：

- 陀螺仪、加速度计和磁力计以 `50Hz` 输入 Mahony filter。
- ICM20948 板载传感器坐标会先映射到 AHRS 使用的右手坐标系，再送入 Mahony，避免静止时姿态解算翻转。
- 2026-06-18 旋转测试 CSV 显示原始 `IGZ` 与裸磁航向变化方向一致；AHRS Z 轴陀螺输入不再取反，避免转动时 Mahony yaw 反向积分、停下后再被磁力计拉回。
- 陀螺仪负责短时间姿态积分，加速度计约束 roll/pitch，磁力计校正 yaw。
- 磁场模长偏离最近稳定参考值过大时，临时退化为 `updateIMU`，也就是只用陀螺仪和加速度计更新，并上报 `IQUAL=MAG_DISTURBED`。
- 检测到正在转动时，磁力计只记录不强拉航向，融合短暂进入 `IQUAL=GYRO_TURNING`；停稳后的短时间恢复窗口进入 `IQUAL=GYRO_RECOVER`，用更快但受限的 yaw offset 校正贴回磁航向，随后再回到慢速稳定校正，减少旋转后的跳变和长时间回弹。
- 输出字段中 `IHDG` 表示 ESP32 Mahony 影子融合航向，`IMHDG` 表示当前磁力计裸航向，`IAHRS` 表示 Mahony 原始 yaw，`IROLL/IPITCH` 用于观察姿态解算。
- 为保持与现有船体航向坐标一致，固件首次拿到有效磁航向时会用 `IMHDG` 对 Mahony yaw 做一次偏移对齐；后续磁场质量正常且转动较小时再慢速修正 yaw offset，避免陀螺积分造成静止单向漂移。
- `IHDG` 输出再经过环形低通滤波；转动时响应更快，静止时更稳。
- IMU 调试测试期间曾临时把 ESP32 蓝牙 `STATUS` 提高到 `10Hz` 并写入 Android CSV；当前等待新 IMU 到货，调试 UI 和 CSV 写入已暂时关闭，`STATUS` 恢复为常规 `1Hz` 上报。
- 2026-06-18 第二轮旋转测试确认 Z 轴方向修复有效，但 `IHDG` 转动中仍比 `IMHDG` 多走约 `8-10°`；因此把转动输出低通从 `0.45` 降到 `0.28`，恢复输出从 `0.32` 降到 `0.20`，恢复期 yaw offset 修正从 `0.060` 降到 `0.035`，优先减少冲过头和肉眼可见回弹。
- 2026-06-18 第三轮测试显示停下后的回弹基本消失，但快速手动旋转时 `IHDG` 仍会被短时陀螺尖峰甩出；因此对输出航向增加速率上限：转动中约 `35°/s`，恢复期约 `22°/s`，静止期约 `8°/s`。该限制只作用于上报的影子航向，不改变 Mahony 内部状态，也不参与推进控制。

这个实现仍然只用于 Android 设置页观测，不参与默认航向锁定、角度转向或自动导航。

### 阶段 4：手机对比和质量评分

Android 同时显示：

- 手机航向。
- ESP32 融合航向。
- 两者差值。
- ESP32 航向质量：`OK / CAL_REQUIRED / MAG_DISTURBED / GYRO_TURNING / GYRO_RECOVER / GYRO_SETTLING / STALE`。

建议通过蓝牙 `STATUS` 新增维护字段：

```text
IHDG=<deg>;IQUAL=<state>;IERR=<phone_minus_imu_deg>
```

其中 `IERR` 可以先由 Android 本地计算，不一定由 ESP32 输出。

### 阶段 5：受控接入

只有满足以下条件，才允许把 ESP32 融合航向作为可选来源：

- 静止 60 秒，航向峰峰值不超过 `2°`。
- 手机同位置对比，稳定环境下平均误差小于 `5°`。
- 低速直线测试中，ESP32 航向变化与 GPS 航迹/手机航向趋势一致。
- 电机不同油门下，磁力计质量不频繁进入 `MAG_DISTURBED`。
- 任意异常时控制逻辑仍回到手机航向或停止，而不是继续用错误航向。

## 标定要求

### 陀螺仪

- 上电后静止采样，计算三轴零偏。
- 如果上电时板体正在移动，应拒绝保存零偏，并提示重新静止。
- 温度变化可能造成漂移，后续可做慢速在线零偏估计，但不能在转弯时误学。

### 加速度计

- 第一阶段可只用单位化重力方向。
- 后续做六面标定，修正零偏和比例误差。

### 磁力计

- 需要从 2D X/Y 校准升级为 3D 校准。
- 最低要求保存三轴 offset 和 scale。
- 正式使用前应在实际安装位置、整套电源/电机线束固定后标定。
- 电机大电流运行时需要记录磁场模长变化，确认是否需要按油门状态降权。

## 安全规则

- 影子航向不得改变 `ARM`、`L/R`、自动导航状态或锁航状态。
- ESP32 融合航向质量不为 `OK` 时，Android 不允许选择它作为控制来源。
- 任何 I2C 读失败、姿态解算异常、磁场异常、陀螺饱和或航向超时，都必须保持当前控制来源不变，并在 UI 提示维护状态。
- 自动导航和航向锁定默认仍使用手机航向，直到完成单独安全评审。

## 测试计划

1. 桌面静止测试：记录 10 分钟，比较裸磁航向、倾斜补偿航向、融合航向和手机航向。
2. 手持慢速旋转测试：每次转 `90°`，检查回到原方向后的误差。
3. 倾斜测试：同一航向下改变 roll/pitch，检查航向是否保持稳定。
4. 电机断电安装位测试：确认实际安装后的静止噪声。
5. 电机低功率运行测试：记录磁场模长和航向跳动。
6. 岸上空载控制测试：只显示差值，不接入控制。
7. 水下低速影子测试：手机控制，ESP32 融合航向仅记录。

## 实施顺序

1. 固件补齐 ICM20948 加速度计和三轴陀螺仪读取。
2. 固件把裸 9 轴数据摘要和当前磁场模长通过维护状态输出。
3. Android 设置页增加“ESP32 IMU 观测”面板。
4. 实现倾斜补偿磁航向，新增 `IHDG_TILT`。
5. 实现 Mahony/Madgwick 融合，新增 `IHDG` 和 `IQUAL`。
6. Android 增加手机/ESP32 航向差值曲线或最近峰峰值。
7. 完成测试记录后，再决定是否允许 ESP32 融合航向作为可选控制来源。
